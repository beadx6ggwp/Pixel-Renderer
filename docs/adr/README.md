# Architecture Decision Records

這裡放 Architecture Decision Records，也就是未來重要工程選擇的短紀錄。

ADR 不需要長。每一篇只回答：

```text
Context
Decision
Consequences
Alternatives
Status
```

建議命名：

```text
0001-owned-framebuffer.md
0002-display-backend-boundary.md
0003-depth-convention.md
0004-screen-space-triangle-first.md
0005-material-vs-material-instance.md
0006-command-queue-after-renderer-skeleton.md
```

## Template

```md
# ADR-0000: Title

## Status

Proposed / Accepted / Superseded

## Context

What problem are we solving?

## Decision

What decision did we make?

## Consequences

What becomes easier?
What becomes harder?
What must stay true?

## Alternatives

What did we reject, and why?
```

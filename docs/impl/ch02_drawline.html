IMPL-02 · Bresenham 直線實作
DrawLine — Bresenham 8 方向

1 完整程式碼一覽

render_device.cpp — RenderDevice::DrawLine()
29void RenderDevice::DrawLine(int x1, int y1, int x2, int y2, uint32_t color) {
30 // Bresenham
31 int dx = std::abs(x2 - x1);
32 int dy = std::abs(y2 - y1);
33 int sx = (x1 < x2) ? 1 : -1;
34 int sy = (y1 < y2) ? 1 : -1;
35 int err = dx - dy;
36 while (true) {
37 SetPixel(x1, y1, color);
38 if (x1 == x2 && y1 == y2) break;
39 int e2 = 2 * err;
40 if (e2 > -dy) { err -= dy; x1 += sx; }
44 if (e2 < dx) { err += dx; y1 += sy; }
48 }
49}

2 第 31–32 行：dx, dy — 絕對距離

dx = |x2 - x1| ← x 方向的總像素跨度
dy = |y2 - y1| ← y 方向的總像素跨度

取絕對值後，dx 和 dy 永遠是正整數，不帶方向資訊。 方向資訊由 sx、sy 單獨處理。這樣做把「距離」和「方向」拆開， 讓後面的 err 邏輯不需要處理正負號問題。

3 第 33–34 行：sx, sy — 步進方向

sx = (x1 < x2) ? +1 : -1 ← 每步 x 往右或往左走
sy = (y1 < y2) ? +1 : -1 ← 每步 y 往下或往上走

sx 和 sy 就是理論 Ch06 中的「符號對稱」機制。 同一套 err 邏輯套上不同的 sx/sy，就能畫出 8 個方向的直線。 例如：sx=1, sy=1 是右下方；sx=-1, sy=1 是左下方。

4 第 35 行：err = dx − dy — 初始誤差

err = dx - dy

err 是 Bresenham 的核心狀態變數，代表「x 方向已累積的誤差減去 y 方向已累積的誤差」。 初始值 dx - dy 來自 d₁d₂ 法的推導（Ch05）：

初始決策參數 P₀ = 2·dy − dx（d₁d₂ 法）
誤差累積法的 err 初始值 = dx − dy
兩者差一個常數移位（乘以 −1 再加 dx），數學上等價——見 Ch07。

5 第 39–44 行：每步決策（兩個獨立的 if）

核心迴圈
39int e2 = 2 * err; // 預乘 2，避免除法
40if (e2 > -dy) { err -= dy; x1 += sx; } // 條件：x 方向前進
44if (e2 < dx) { err += dx; y1 += sy; } // 條件：y 方向前進
為什麼是兩個獨立的 if，不是 if-else？
對角線方向（例如 45°）同一步需要同時更新 x 和 y。 如果用 if-else，對角線就會漏掉一個軸的更新，畫出來的線會偏斜。 兩個獨立 if 允許同一步「x 和 y 都更新」——這正是對角線情況。

e2 = 2 * err
避免小數：原本的條件是 err > -dy/2 和 err < dx/2， 兩邊乘 2 消分母，全程整數運算。

err 的更新邏輯

x 前進時：err -= dy ← 「x 走了一步，但理想線只需要 dy/dx 個 y 單位，補償不足」
y 前進時：err += dx ← 「y 升一格，累積正誤差，等待下次 x 前進消耗」

你可以把 err 想成一個「y 欠債計數器」： 每走一步 x 就欠 dy 份 y， 每升一格 y 就還 dx 份。 當欠債超過門檻就升格（y 前進），低於門檻就繼續走 x。 這正是 Ch04 誤差累積法的核心直覺。

6 終止條件：抵達終點就停

迴圈內
37SetPixel(x1, y1, color); // 先畫當前點
38if (x1 == x2 && y1 == y2) break; // 再判斷是否到終點

「先畫後判斷」的順序確保終點也被畫到。 如果先判斷再畫，當起點 == 終點（單點線段）時就不會畫任何東西。

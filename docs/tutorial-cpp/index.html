▌ 這份教學的定位

這不是「從零學 C++ 語法」，而是把你曾經理解過的 C++ 語意重新接回來。每章會先問一個工程問題： 為什麼 C++ 需要這個機制？它解決了 C 的哪個語意缺口？如果要做 renderer、framebuffer、resource、UI 或 backend abstraction， 這個機制到底幫你保證了什麼？

Part 0-1
程式如何被建構、物件如何存在、型別如何建立語意邊界
Part 2-3
把數學型別、資源擁有權、泛型演算法放進 renderer 設計裡
Part 4-5
把架構邊界、numeric policy、performance、UI/shader、verification 串成工程路線

▌ 章節寫法

章節會依照你的 Obsidian vault 裡的 writing principles：先從具體現象或小段 code 開始， 讓 naive model 失敗，再往下拆 execution / memory / object lifetime，往上收束到 semantic rule、invariant 與 design tradeoff。 最後必須回到 implementation 與 verification，而不是停在名詞定義。

Phenomenon
先看一段會讓人疑惑的 code 或 renderer 場景
Naive Model
明確展示直覺模型哪裡不夠精準或會出錯
Semantic Rule
再補正式語意、invariant、cost model 與可驗證方式

▌ 思考深度判準

後續章節不應被固定成同一種模板。I-04 的價值不是形式，而是思考深度：不能只列 API、語法或配置步驟。 每章要依主題選擇自己的鋪陳，但必須讓 naive model 被檢驗，往下拆到必要的 machine / memory / object / process / ABI / build graph 等機制， 再回到 C++ 語意或工程規則，最後落到 Pixel-Renderer 的設計後果與可驗證方法。

Mechanism
先問底層實際發生什麼：compiler transform、OS process、debug trap、object lifetime、buffer ownership
Semantic Rule
再收束成 C++ 語意：type/value category、RAII、invariant、contract、generic requirement
Verification
最後給出能在專案裡檢查的東西：GDB 指令、build command、設定檔、refactor checklist

▌ 前向編排

這份入口不再保留舊線性章節作為平行主線。統一版會分成六個 Part：先理解 program/binary，再理解 object/lifetime，再處理 value transfer 和泛型合約， 最後才進入 renderer architecture、UI/shader boundary、low-level performance。

Foundation
T-01 到 T-05 加上 T-22/T-23/T-24/T-25/T-26 建立 binary、type、object lifetime、value category、const/cv、overload/operator、deduction、static/global state、regular/equality/hash、copy/move 的地基
Engineering
I-02 到 I-07 先把 toolchain、debugger、VSCode、Make、CMake 整理成可維護工作流
Architecture
I-08 之後才進入 Framebuffer、DisplayBackend、Win32/SDL/macOS、UI 與 shader interface

▌ 服務的專案擴充方向

這份 C++ 複習最終要服務 Pixel-Renderer 的下一階段，而不是停在語言本身。每個概念都要能回到 renderer 的設計問題： 誰擁有 framebuffer？shader object 如何被複製或禁止複製？UI state 放在哪裡？Win32 / SDL / macOS backend 如何替換？ render frontend 與 backend 的邊界應該用 value、reference、pointer、virtual interface，還是 template？

Render Backend
owned framebuffer、DisplayBackend、Win32/SDL/macOS present path 的 lifetime 與 ownership
Shader Pipeline
shader interface、vertex/fragment data、copy/move policy、polymorphism cost
Immediate UI
UI command buffer、widget state、resource cache、renderer-facing draw data 的型別設計

▌ 統一版主線：兩條學習線

統一版拆成兩條線：C++ 深度複習 負責把語言本身講完整，Pixel-Renderer 工程實作 負責處理抽象分離、跨平台編譯、VSCode 配置與 backend refactor。

Theory Track
C++ 深度複習：語意、型別、效能
這條線不應只停在 move。它要完整補齊 C++ 對程式存在方式、object model、ownership adapters、value category、generic programming、STL、error handling、low-level performance 的規範。
T-01 Compilation / Linkage / ABI完成
declaration vs definition、ODR、translation unit、name mangling、static/extern、library linking、MSVC/GCC/Clang ABI 差異。
T-02 Type System / Initialization完成
implicit conversion、explicit、enum class、const/constexpr、brace initialization、narrowing、aggregate 與 uniform initialization 的坑。
T-03 Object Model / Storage / Lifetime / UB完成
storage duration、object lifetime、placement new、alignment、strict aliasing、object representation、trivial/standard-layout、undefined behavior。
T-18 Casts / Type Punning / Bit Cast完成
static_cast、const_cast、dynamic_cast、reinterpret_cast、std::bit_cast、strict aliasing、alignment、bytes 與 object representation。
T-04 Value Category 完整版完成
lvalue/xvalue/prvalue、temporary materialization、lifetime extension、reference collapsing、forwarding reference、std::forward、operator return category。
T-22 Const / cv-qualification / Value Category完成
const T、const T&&、move-from-const、member function const、mutable、ref-qualified member function，把 value category 接到 mutation capability。
T-23 Overload / Conversion / Operator Design完成
overload resolution、explicit conversion、operator design、hidden friend、ADL、equality/hash contract，服務 math type、strong id、cache key。
T-24 Type Deduction / auto / decltype / Forwarding完成
auto、auto&、const auto&、auto&&、decltype、decltype(auto)、template deduction、forwarding reference，避免 command/view/generic helper 失去 reference 語意。
T-25 Static Storage / Global State / Initialization Order完成
static storage duration、global state、initialization order、function-local static、thread_local、composition root，避免 renderer state 變成 hidden lifetime。
T-26 Regular Types / Equality / Hashing / Ordering完成
Regular/Semiregular、equality laws、hash consistency、strict weak ordering、float pitfalls、cache key、command diff 與 golden test equality policy。
T-05 Copy / Move / Object Transfer完成
copy、move、moved-from state、copy elision、return by value、noexcept move，整理 object 如何被交付與轉移。
T-06 RAII / Resource Ownership完成
destructor、Rule of Zero/Five、unique_ptr/shared_ptr/custom deleter、exception safety、move-only resource handle。
T-17 Smart Pointer / Ownership Adapters完成
unique_ptr、shared_ptr、weak_ptr、control block、custom deleter、observer/span、resource handle，避免用 shared_ptr 掩蓋 ownership 問題。
T-07 Callable / Overload / Lambda完成
function pointer、member function pointer、functor、lambda capture lifetime、std::function、overload resolution、default argument、inline function。
T-08 Templates / Concepts / Type Traits完成
function/class template、specialization、SFINAE、concepts、constexpr if、CRTP、policy-based design，連到 shader/math type 泛型化。
T-09 STL / Containers / Iterators完成
vector/array/span/string_view、iterator invalidation、erase-remove、contiguous storage、allocator basics，連到 command buffer 和 vertex buffer。
T-10 Error Handling / Contracts完成
assert、optional、expected-like pattern、exception、noexcept、error code、precondition/postcondition、type invariant。
T-11 Runtime Polymorphism Alternatives完成
virtual、type erasure、variant、visitor、command buffer、template dispatch 的取捨，避免把所有抽象都塞進 vtable。
T-12 Low-level Performance / Memory Layout完成
cache locality、alignment、padding、branch prediction、SIMD intro、AoS vs SoA、memory bandwidth，連到 rasterizer hot loop。
T-13 Concurrency / Memory Model完成
thread、mutex、atomic、data race、producer-consumer、job system intro；先理解規則，再談 tile-based parallel rasterization。
T-14 Modern C++ Boundaries完成
哪些 C++20/23 特性值得提前知道：span、concepts、expected、modules、ranges、coroutine；哪些先不用深入。
T-15 Numeric Semantics / Pixel Formats完成
fixed-width integer、signed/unsigned overflow、bit shift、packed RGBA/BGRA、endianness、float precision、deterministic comparison。
T-16 API Design Conventions完成
owner/view/borrow、const-correctness、reference/pointer/span、return value、out parameter、error return、namespace/header boundary。
Implementation Track
Pixel-Renderer 工程實作：抽象、工具鏈、跨平台
這條線不重講 C++ 語法，而是把 theory 用在專案演進：先讀現有架構，再處理 compiler/toolchain、ownership、backend、verification 與 header boundary。
I-01 Current Renderer Architecture Walkthrough完成
從目前 Application、ScreenManager、RenderDevice、Rasterizer 讀出 object graph、frame flow、CPU framebuffer 與 Win32 present coupling。
I-02 Toolchain 地圖完成
MSVC、MinGW GCC、Apple Clang、standard library、ABI、debug symbols、compiler flags，以及 OpenGL/D3D/SDL/ImGui 這類 library 的接入層次。
I-03 Build Workflow：Compiler 到 CMake完成
從手動 compiler/linker、VSCode task/launch、GDB/LLDB、makefile 工作流，循序走到 CMake target graph。
I-04 Debugger Fundamentals: GDB / LLDB完成
process、breakpoint、stack frame、debug symbols、memory inspect、watchpoint，以及 Pixel-Renderer debug recipes。
I-05 VSCode 配置完成
tasks.json、launch.json、c_cpp_properties.json、CMake Tools、Windows/macOS debug workflow，處理 clang/gcc include path 問題。
I-06 Make Workflow完成
從手動編譯命令走到 make rule、dependency、debug/release target，理解目前 makefile 的邊界。
I-07 CMake Target-based Build完成
target、include path、compile definitions、platform-specific source、library linking，避免用全域設定堆出不可維護 build。
I-19 Preprocessor / Compile-time Configuration完成
整理 include、macro、platform macro、feature macro、conditional compilation、target_compile_definitions，避免 renderer core 被 #ifdef 污染。
I-20 Dependency Management / Third-party Integration完成
把 SDL、OpenGL、D3D、Metal、ImGui 拆成 include、compile definition、link library、runtime artifact、package discovery 與 target boundary。
I-08 Framebuffer ownership refactor完成
抽出 owned Framebuffer，讓 RenderDevice 借用它，而不是借用 ScreenManager 的 Win32 DIB memory。
I-09 DisplayBackend Interface完成
定義 window/event/present 的替換點，讓 Win32/SDL/macOS backend 只接收 const Framebuffer& 做 present。
I-10 Win32DisplayBackend完成
把目前 ScreenManager 的 DIBSection、memory DC、window DC、message pump 與 BitBlt 落到 Win32 backend present path。
I-11 SDLDisplayBackend完成
整理 SDL2/3、SDL_Renderer backend、UpdateTexture/LockTexture、OS compositor，最後收束到 SDL present backend 的實作邊界。
I-12 Cross-platform Development Path完成
以 Windows / macOS / Ubuntu 為主軸，整理 toolchain、debugger、SDL dependency、CMake preset、VSCode 與 backend selection。
I-13 Renderer Frontend / Backend Boundary完成
用架構地圖分清 rendering frontend、software rendering backend、owned framebuffer、display backend，以及 UI/ImGui/shader 未來接入層。
I-14 Immediate-mode UI Integration Boundary完成
整理自製 UI / ImGui draw data 如何進入 software renderer：widget state、layout、draw command、font atlas、clip rect、alpha blend。
I-15 Shader Pipeline Interface Boundary完成
把 future shader pipeline 的資料邊界、hot-loop dispatch、vertex/fragment input/output 與 command buffer interface 分清楚。
I-16 Verification / Golden Image / Regression Testing完成
建立 headless framebuffer tests、golden image、debug dump、exact/tolerance policy，讓 renderer core 的 correctness 不依賴肉眼看 window。
I-17 Header Boundary / Dependency Hygiene完成
控制 public/private header、platform type、include/compile/link dependency，防止 windows.h、SDL.h 污染 renderer core。
I-18 Diagnostics / Sanitizers / Static Analysis完成
把 warnings、ASan、UBSan、static analysis、debug/sanitized/release profile 接到 framebuffer、numeric semantics、lifetime 與 golden tests。

▌ Part Dependency Map

這份教學的閱讀順序不是「先 C++、再工程」的單向排列，而是每個 Part 都建立下一層需要的 invariant。 Part 0 先讓你知道程式如何變成 process；Part 1 到 Part 3 建立 object、value、generic contract；Part 4 才談 renderer abstraction； Part 5 則把效能、numeric policy、UI/shader、verification、header hygiene 接成未來擴充路線。

SVG：這不是章節數量圖，而是 dependency graph。若某層語意沒建立，後面的 abstraction 會變成只有 class 名稱。
Part 0 output
你能分辨 source、object file、symbol、library、preprocessor、third-party dependency、debugger、sanitizer、build profile 各自處理哪一層。
Part 1 output
你能判斷 resource owner、borrowed view、object lifetime、object representation，並知道 smart pointer、cast、RAII wrapper、resource handle 各自跨越哪種邊界。
Part 2 output
你能看懂 reference binding、const/cv qualification、overload/conversion/operator、type deduction、static/global lifetime、regular/equality/hash、move-from-const、copy/move、lambda capture、operator return category 對 renderer data lifetime 的影響。
Part 3 output
你能把 type requirement、container invalidation、contract、invariant 轉成 command buffer / vertex buffer 設計規則。
Part 4 output
你能明確說出 renderer core、software rendering backend、DisplayBackend、Win32/SDL/macOS/Ubuntu layer 的替換邊界。
Part 5 output
你能把 cache、numeric policy、UI draw data、shader interface、golden tests、header hygiene 串成可演進的工程路線。

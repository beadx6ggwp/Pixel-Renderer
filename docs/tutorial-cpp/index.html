▌ 這份教學的定位

這不是「從零學 C++ 語法」，而是把你曾經理解過的 C++ 語意重新接回來。每章會先問一個工程問題： 為什麼 C++ 需要這個機制？它解決了 C 的哪個語意缺口？如果要做 renderer、framebuffer、resource、UI 或 backend abstraction， 這個機制到底幫你保證了什麼？

Part 1-2
程式如何被建構、物件如何存在、型別如何建立語意邊界
Part 3-4
把數學型別、資源擁有權、泛型演算法放進 renderer 設計裡
Part 5-6
回到 Modern C++ 的核心：RAII、smart pointer、move semantics、abstraction cost

▌ 章節寫法

章節會依照你的 Obsidian vault 裡的 writing principles：先從具體現象或小段 code 開始， 讓 naive model 失敗，再往下拆 execution / memory / object lifetime，往上收束到 semantic rule、invariant 與 design tradeoff。 最後必須回到 implementation 與 verification，而不是停在名詞定義。

Phenomenon
先看一段會讓人疑惑的 code 或 renderer 場景
Naive Model
明確展示直覺模型哪裡不夠精準或會出錯
Semantic Rule
再補正式語意、invariant、cost model 與可驗證方式

▌ 思考深度判準

後續章節不應被固定成同一種模板。I-04 的價值不是形式，而是思考深度：不能只列 API、語法或配置步驟。 每章要依主題選擇自己的鋪陳，但必須讓 naive model 被檢驗，往下拆到必要的 machine / memory / object / process / ABI / build graph 等機制， 再回到 C++ 語意或工程規則，最後落到 Pixel-Renderer 的設計後果與可驗證方法。

Mechanism
先問底層實際發生什麼：compiler transform、OS process、debug trap、object lifetime、buffer ownership
Semantic Rule
再收束成 C++ 語意：type/value category、RAII、invariant、contract、generic requirement
Verification
最後給出能在專案裡檢查的東西：GDB 指令、build command、設定檔、refactor checklist

▌ 前向編排

接下來不會把舊 Ch01-Ch18 當成最終線性順序，也不會只沿著 implementation 一路寫到 backend。 新版會分成六個 Part：先理解 program/binary，再理解 object/lifetime，再處理 value transfer 和泛型合約， 最後才進入 renderer architecture、UI/shader boundary、low-level performance。

Foundation

Engineering

Architecture
I-08 之後才進入 Framebuffer、DisplayBackend、Win32/SDL/macOS、UI 與 shader interface

▌ 服務的專案擴充方向

這份 C++ 複習最終要服務 Pixel-Renderer 的下一階段，而不是停在語言本身。每個概念都要能回到 renderer 的設計問題： 誰擁有 framebuffer？shader object 如何被複製或禁止複製？UI state 放在哪裡？Win32 / SDL / macOS backend 如何替換？ render frontend 與 backend 的邊界應該用 value、reference、pointer、virtual interface，還是 template？

Render Backend
owned framebuffer、DisplayBackend、Win32/SDL/macOS present path 的 lifetime 與 ownership
Shader Pipeline
shader interface、vertex/fragment data、copy/move policy、polymorphism cost
Immediate UI
UI command buffer、widget state、resource cache、renderer-facing draw data 的型別設計

▌ 下一階段：拆成兩條學習線

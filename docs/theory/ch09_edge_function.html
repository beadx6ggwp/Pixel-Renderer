PART 3 · CHAPTER 09
邊緣函數（Edge Function）
把叉積用在三條邊上，判斷點是否在三角形內
edge function inside test winding order tie-breaking rule

上一章我們建立了一個工具：cross(AB, AP) 的正負號告訴你 P 在 AB 的哪一側。

現在把這個工具用在三角形的三條邊上——這就是 Edge Function。

1
核心概念：三條邊，三次測試

給定三角形頂點 V0、V1、V2，它有三條邊：

$\begin{array}{ll} \text{邊 0：} & V_0 \rightarrow V_1 \\[4pt] \text{邊 1：} & V_1 \rightarrow V_2 \\[4pt] \text{邊 2：} & V_2 \rightarrow V_0 \end{array}$

對每條邊，做一次 Ch08 的 edgeTest()：

$\begin{array}{l} E_0(\mathbf{P}) = \text{cross}(\mathbf{V}_1 - \mathbf{V}_0,\; \mathbf{P} - \mathbf{V}_0)\\[6pt] E_1(\mathbf{P}) = \text{cross}(\mathbf{V}_2 - \mathbf{V}_1,\; \mathbf{P} - \mathbf{V}_1)\\[6pt] E_2(\mathbf{P}) = \text{cross}(\mathbf{V}_0 - \mathbf{V}_2,\; \mathbf{P} - \mathbf{V}_2) \end{array}$

這三個值就是三角形的 邊緣函數（Edge Function）。

判斷法則：
如果 E0(P)、E1(P)、E2(P) 三個值同號（都 ≥ 0 或都 ≤ 0）， 則 P 在三角形內（或邊上）。

2
繞向（Winding Order）決定「哪個符號算通過」

注意「同號」這個條件——是都正，還是都負？ 這取決於三角形頂點的排列順序，也就是繞向（Winding Order）。

▌ 逆時針（CCW）

V0 → V1 → V2 逆時針排列
三個 Edge Function 對內部點都是正值

數學座標系（y 朝上）的慣例。
OpenGL 預設使用 CCW 為正面。

▌ 順時針（CW）

V0 → V1 → V2 順時針排列
三個 Edge Function 對內部點都是負值

螢幕座標系（y 朝下）常見。
DirectX 預設使用 CW 為正面。

實作建議：統一標準，不要混用。
一種做法：把三個值都取絕對值再判正負——但這樣無法判斷背面。
更好的做法：先用 signedArea 知道繞向，再決定「>= 0 還是 <= 0」算通過。
或者更簡單：直接要求三個值同號即可，不在乎繞向。

用程式碼表達

// 判斷 P 是否在三角形 (V0, V1, V2) 內
bool insideTriangle(Vec2 V0, Vec2 V1, Vec2 V2, Vec2 P) {
    float e0 = edgeTest(V0, V1, P);
    float e1 = edgeTest(V1, V2, P);
    float e2 = edgeTest(V2, V0, P);

    // 三個值同號（都 >= 0 或都 <= 0）即為在三角形內
    return (e0 >= 0 && e1 >= 0 && e2 >= 0)
        || (e0 <= 0 && e1 <= 0 && e2 <= 0);
}

3
邊緣函數是線性函數

Edge Function 有一個非常重要的性質：它是 P 的線性函數。

把 E0 展開：

$E_0(\mathbf{P}) = (V_{1x}-V_{0x})(P_y-V_{0y}) - (V_{1y}-V_{0y})(P_x-V_{0x})$

令 \(a = V_{1y}-V_{0y}\)，\(b = -(V_{1x}-V_{0x})\)，整理後：

$E_0(x,\,y) = \textcolor{#a6e3a1}{a \cdot x + b \cdot y + c} \qquad\text{（一次多項式；a, b, c 為常數，只依賴頂點位置）}$

這意味著：當 x 增加 1，E0 增加固定量 a；當 y 增加 1，E0 增加固定量 b。

增量更新（Incremental Update）

假設你已經算出像素 (x, y) 的 E0 值，那麼：

E0(x+1, y) = E0(x, y) + a
E0(x, y+1) = E0(x, y) + b

移到下一個像素，只需要一次加法——不需要重新算叉積！
GPU 硬體的光柵化單元正是用這個技巧，以常數時間處理每個像素。

4
完整的光柵化結構

把 Edge Function 接上掃描框架，就得到完整的三角形填色演算法：

1
計算 a、b、c（三條邊各一組）
只需做一次，與像素數量無關。
2
找包圍盒（下一章詳述）
限制要測試的像素範圍，避免掃描整張畫面。
3
掃描包圍盒，每列用增量更新
每換一列，E 值重新從該列左端算起（加 b）；每換一行，加 a。
4
三個 E 同號 → 點亮像素
並記錄 E0、E1、E2 值（下一章用來算重心座標）。
// 計算 E0 的增量係數
// E0(x,y) = a0*x + b0*y + c0
float a0 = V1.y - V0.y;
float b0 = V0.x - V1.x;
float c0 = V1.x*V0.y - V0.x*V1.y;
// （同樣計算 a1,b1,c1 和 a2,b2,c2）

// 掃描包圍盒（[xmin,xmax] × [ymin,ymax]）
for (int y = ymin; y <= ymax; y++) {
    // 每列開頭重新計算三個 E 的起始值
    float e0 = a0*xmin + b0*y + c0;
    float e1 = a1*xmin + b1*y + c1;
    float e2 = a2*xmin + b2*y + c2;

    for (int x = xmin; x <= xmax; x++) {
        if ((e0 >= 0 && e1 >= 0 && e2 >= 0)
        || (e0 <= 0 && e1 <= 0 && e2 <= 0)) {
            SetPixel(x, y, color);
        }
        // 增量更新：x+1 → 每個 E 加對應的 a
        e0 += a0; e1 += a1; e2 += a2;
    }
}

5
邊界像素的歸屬：Tie-breaking Rule

如果兩個三角形共享一條邊，這條邊上的像素可能被兩個三角形都聲稱「通過測試」 （E = 0，剛好在邊上）。這會導致重複渲染，產生閃爍或混色。

解法是 Tie-breaking Rule（也叫 Top-Left Rule）：

Top-Left Rule（OpenGL / DirectX 標準）

只有當邊是「頂邊（top edge）」或「左邊（left edge）」時，才把 E = 0 的像素算作通過。

頂邊：水平邊，且在三角形的上方（螢幕座標 y 最小）。
左邊：非水平邊，且往下走（dy > 0，螢幕座標）。

其他方向的邊遇到 E = 0 時，用嚴格不等式（> 0 而非 >= 0）排除該像素。

這樣每個共用邊上的像素恰好只屬於一個三角形，不重不漏。

實作上，可以對每條邊的判斷條件加上微調：

// 判斷這條邊是否為 top edge 或 left edge
bool isTopLeft(Vec2 A, Vec2 B) {
    Vec2 edge = { B.x - A.x, B.y - A.y };
    bool isTop = (edge.y == 0 && edge.x > 0); // 水平且向右
    bool isLeft = (edge.y < 0); // 向上（螢幕 y 向下，所以 dy<0 = 向上）
    return isTop || isLeft;
}

// 測試時：top/left edge 用 >=0，其他用 >0
float bias0 = isTopLeft(V0, V1) ? 0 : -1;
float bias1 = isTopLeft(V1, V2) ? 0 : -1;
float bias2 = isTopLeft(V2, V0) ? 0 : -1;

if (e0 + bias0 >= 0 && e1 + bias1 >= 0 && e2 + bias2 >= 0) {
    SetPixel(x, y, color);
}
實作上的優先順序：
初學實作時，可以先不做 Tie-breaking Rule——只是偶爾共用邊會有一個像素重複畫， 視覺上幾乎看不出來。
等到整個管線跑起來再回來加這個細節。

6
與 GPU 的連結

你剛才看到的結構，就是現代 GPU 光柵化單元的核心邏輯：

GPU 光柵化（簡化版）
1. 頂點 Shader → 得到三角形頂點的螢幕座標
2. 計算三條邊的 Edge Function 係數（a, b, c）
3. 找包圍盒
4. 以 2×2 像素 Quad 為單位，同時測試四個像素
5. 通過的像素送入片段 Shader
6. Top-Left Rule 處理共用邊

第 4 步的 2×2 Quad 是 GPU 特有的設計—— 同時處理四個相鄰像素，是為了讓 fragment shader 可以用差分計算偏微分 （dFdx、dFdy，用於 mipmap 選層）。 Quad 中即使有像素在三角形外面，也會執行 shader（只是結果被丟棄）， 這就是為什麼很細的三角形特別浪費 GPU。

7
本章小結

本章學到的三件事：

1. Edge Function = 叉積用在三條邊上，三值同號 → 點在三角形內。
2. Edge Function 是線性函數，可以用增量加法（a / b）避免重複計算。
3. 共用邊用 Top-Left Rule 決定歸屬，保證每個像素只被渲染一次。

目前我們只是知道「哪些像素在三角形內」，還不知道要填什麼顏色。 下一章：重心座標（Barycentric Coordinates）—— Edge Function 算出的 E0、E1、E2 值，可以直接轉換成插值係數， 用來在三角形內插值任何頂點屬性（顏色、UV、法線…）。

PART 7 · CHAPTER 25
IShader 介面設計
用 C++ 抽象類別模擬 GLSL 的 vertex/fragment shader——Uniforms、Varyings、以及讓不同著色器能替換的設計
IShader uniforms varyings 抽象類別 FlatShader

Ch24 確認了我們需要兩個「鉤子」：vertex() 做 MVP 變換並輸出 Clip Space 座標， fragment() 決定每個像素的最終顏色。 本章的任務是把這兩個鉤子設計成一個乾淨的 C++ 抽象介面 IShader， 讓不同的著色效果（Flat、Gouraud、Phong、Texture…）都能透過繼承來實作， 而光柵化迴圈本身不需要任何修改。

1
設計目標

我們要達成的效果是這樣的——光柵化迴圈寫一次，著色效果可以任意抽換：

// 光柵化器：不知道也不在乎「用什麼方法著色」 FlatShader shader_a; GouraudShader shader_b; PhongShader shader_c; device->DrawTriangle(shader_a, face_idx); // 純色填充 device->DrawTriangle(shader_b, face_idx); // 頂點光照插值 device->DrawTriangle(shader_c, face_idx); // 逐像素光照

三個 DrawTriangle 呼叫——光柵化的包圍盒掃描、重心座標計算、Z-buffer 測試—— 一行都沒有改。只有 vertex() 和 fragment() 的實作換掉了。 這就是多型（Polymorphism）的威力，也是 GPU Shader 架構的本質。

2
Uniform vs Varying：兩種資料的職責

在設計介面之前，必須先弄清楚資料的流向。GLSL 把 Shader 內的資料分成三類：

類型 生命週期 誰寫入 誰讀取 典型範例
Uniform 整個 DrawCall 不變 CPU（在呼叫前設定） vertex() 和 fragment() 都可讀 MVP 矩陣、光照方向、貼圖
Varying 逐頂點寫、逐像素讀 vertex() 寫入 fragment() 讀取（插值後） UV 座標、世界法向量、頂點顏色
Attribute 逐頂點獨有 CPU（在 VBO 裡） vertex() 讀取 物件空間位置、原始 UV、原始法向量

在我們的軟體光柵化中，Attribute 等同於「從 Mesh 資料結構裡取出的頂點資料」， Uniform 是 Shader 的成員變數（在 DrawCall 前設定好）， Varying 也是 Shader 的成員變數（由 vertex() 寫入，由 fragment() 讀取）。

C++ 的映射： GLSL 把 Uniform 和 Varying 用 layout 關鍵字聲明在 Shader 程式的全域範圍； 在 C++ 裡，我們直接把它們寫成 Shader 子類別的成員變數。 這樣每個 Shader 實例都有自己的 Uniform 和 Varying 儲存空間， 而光柵化器透過介面指標呼叫，完全不需要知道內部結構。

3
需要的數學型別

設計 IShader 之前，我們需要幾個基本數學型別。 現有的 Vec3f 只有三個分量，但 Clip Space 需要四分量齊次座標 \((x, y, z, w)\)， MVP 矩陣運算需要 4×4 矩陣。

// ── 新增到 render_device.h（或獨立的 math.h）── struct Vec2f { float x, y; Vec2f(float x = 0, float y = 0) : x(x), y(y) {} }; struct Vec4f { float x, y, z, w; Vec4f(float x=0, float y=0, float z=0, float w=1) : x(x),y(y),z(z),w(w) {} Vec4f(Vec3f v, float w=1) : x(v.x),y(v.y),z(v.z),w(w) {} }; struct Mat4 { float m[4][4] = {}; // 行主序（row-major） Vec4f operator*(const Vec4f& v) const { return { m[0][0]*v.x + m[0][1]*v.y + m[0][2]*v.z + m[0][3]*v.w, m[1][0]*v.x + m[1][1]*v.y + m[1][2]*v.z + m[1][3]*v.w, m[2][0]*v.x + m[2][1]*v.y + m[2][2]*v.z + m[2][3]*v.w, m[3][0]*v.x + m[3][1]*v.y + m[3][2]*v.z + m[3][3]*v.w }; } Mat4 operator*(const Mat4& o) const; // 矩陣×矩陣（實作略） static Mat4 identity(); static Mat4 lookAt(Vec3f eye, Vec3f center, Vec3f up); // Ch19 static Mat4 perspective(float fov, float aspect, float near, float far); // Ch18 };

4
IShader 介面定義

有了型別之後，IShader 的定義非常簡潔：

// ishader.h #pragma once #include "math.h" // Vec2f, Vec3f, Vec4f, Mat4 #include <cstdint> struct IShader { virtual ~IShader() = default; // ── Vertex Stage ────────────────────────────────────────────── // 每個三角形呼叫三次（face_idx = 第幾個三角形，vert_idx = 0/1/2） // 必須：從 Mesh 讀取 Attributes，進行 MVP 變換 // 必須：把要傳給 fragment() 的資料存進自己的 Varying 成員 // 回傳：頂點在 Clip Space 的齊次座標（尚未做透視除法） virtual Vec4f vertex(int face_idx, int vert_idx) = 0; // ── Fragment Stage ──────────────────────────────────────────── // 每個通過深度測試的像素呼叫一次 // bar：透視修正後的重心座標（用來插值 Varyings） // color：[out] 要寫入 framebuffer 的像素顏色 // 回傳 false → discard（捨棄此像素，不寫入） virtual bool fragment(Vec3f bar, uint32_t& color) = 0; };
為什麼 fragment() 回傳 bool？ 這對應 GLSL 的 discard 關鍵字——讓 Fragment Shader 可以「放棄」這個像素。 常見用途：Alpha 測試（半透明貼圖的邊緣裁切）、輪廓線繪製（只保留邊界像素）。 回傳 false 時，光柵化器跳過寫入，也不更新 Z-buffer。

6
對照 GLSL：C++ vs GLSL 的映射

GLSL（GPU 上執行）
// Vertex Shader uniform mat4 uMVP; uniform vec3 uLightDir; in vec3 aPos; in vec2 aUV; out vec2 vUV; out float vLight; void main() { gl_Position = uMVP * vec4(aPos,1); vUV = aUV; vLight = dot(aNormal, uLightDir); } // Fragment Shader in vec2 vUV; in float vLight; uniform sampler2D uTex; void main() { vec3 c = texture(uTex, vUV).rgb; gl_FragColor = vec4(c*vLight, 1); }
C++ IShader（CPU 上執行）
struct GouraudShader : IShader { // === Uniforms（成員變數，DrawCall 前設定） Mat4 mvp; Vec3f light_dir; Texture* tex; Mesh* mesh; // === Varyings（vertex 寫，fragment 讀） Vec2f v_uv[3]; // 三頂點的 UV float v_light[3]; // 三頂點的光照強度 Vec4f vertex(int fi, int vi) override { Vec3f pos = mesh->pos(fi, vi); Vec3f n = mesh->normal(fi, vi); v_uv[vi] = mesh->uv(fi, vi); v_light[vi] = dot(n, light_dir); return mvp * Vec4f(pos, 1); } bool fragment(Vec3f bar, uint32_t& c) override { Vec2f uv = bar.interp(v_uv); float lt = bar.interp(v_light); c = tex->sample(uv) * lt; return true; } };

兩邊的結構完全對應：GLSL 的 uniform = C++ 的成員變數（DrawCall 前設定）， GLSL 的 out/in varying = C++ 的陣列成員變數（vertex 寫、fragment 讀）。 關鍵差異是，GLSL 的插值由 GPU 硬體自動完成；在 C++ 版本裡， 插值這件事由光柵化器在呼叫 fragment() 前計算好並傳入 bar（重心座標）。

7
FlatShader：最簡單的完整實作

FlatShader 是驗證整個管線最好的起點：每個三角形填充固定顏色， 不需要貼圖、不需要法向量插值，只需要 MVP 變換是否正確。

struct FlatShader : IShader { // ── Uniforms ──────────────────────────────────────────────── Mat4 mvp; // Model-View-Projection 矩陣 Vec3f light_dir; // 光照方向（世界空間，已正規化） Mesh* mesh; // ── Varyings ──────────────────────────────────────────────── float v_intensity; // 面法向量與光照的點積（整個面相同） // ── vertex() ──────────────────────────────────────────────── Vec4f vertex(int fi, int vi) override { Vec3f pos = mesh->pos(fi, vi); // 只在第 0 個頂點計算面法向量（整個面只需計算一次） if (vi == 0) { Vec3f a = mesh->pos(fi, 0); Vec3f b = mesh->pos(fi, 1); Vec3f c = mesh->pos(fi, 2); Vec3f face_n = normalize(cross(b - a, c - a)); v_intensity = dot(face_n, light_dir); } return mvp * Vec4f(pos, 1.0f); // → Clip Space } // ── fragment() ────────────────────────────────────────────── bool fragment(Vec3f bar, uint32_t& color) override { if (v_intensity < 0.0f) return false; // 背面：discard uint8_t g = (uint8_t)(v_intensity * 255); color = (g << 16) | (g << 8) | g; // 灰階（白光 × 強度） return true; } };

使用方式：

FlatShader shader; shader.mesh = &my_mesh; shader.mvp = Mat4::perspective(...) * Mat4::lookAt(...) * model_matrix; shader.light_dir = normalize({0, 1, 1}); for (int fi = 0; fi < my_mesh.face_count(); fi++) { device->DrawTriangle(shader, fi); }
FlatShader 的驗證價值： 如果 FlatShader 能正確顯示模型的輪廓、透視感、正確的深度排序—— 就代表 MVP 管線、透視除法、Viewport 變換、Z-buffer 全部正確。 在開發 GouraudShader 或 PhongShader 之前，一定要先讓 FlatShader 完全正確。

8
Varying 的插值：bar 的意義

fragment() 收到的 bar 是透視修正後的重心座標 \((\lambda_1, \lambda_2, \lambda_3)\)， \(\lambda_1 + \lambda_2 + \lambda_3 = 1\)。 任何 Varying 的插值只需要一行：

// UV 插值 Vec2f uv = { bar.x * v_uv[0].x + bar.y * v_uv[1].x + bar.z * v_uv[2].x, bar.x * v_uv[0].y + bar.y * v_uv[1].y + bar.z * v_uv[2].y }; // 光照強度插值 float intensity = bar.x*v_light[0] + bar.y*v_light[1] + bar.z*v_light[2]; // 法向量插值（Phong Shading 用） Vec3f N = normalize({ bar.x*v_n[0].x + bar.y*v_n[1].x + bar.z*v_n[2].x, bar.x*v_n[0].y + bar.y*v_n[1].y + bar.z*v_n[2].y, bar.x*v_n[0].z + bar.y*v_n[1].z + bar.z*v_n[2].z });

為什麼是「透視修正後」？ 單純用螢幕座標計算的重心座標做插值，在透視投影下會得到錯誤的結果 （UV 扭曲、法向量不準確）。 光柵化器在傳 bar 給 fragment() 之前，必須做透視修正——這正是 Ch26 的主題。

9
常見的 IShader 子類別一覽

Shadervertex() 做什麼fragment() 做什麼Varyings
FlatShader MVP 變換，計算面法向量 整面填充固定顏色（強度 = N·L） float intensity
GouraudShader MVP 變換，計算每個頂點 N·L 插值三頂點的光照強度 float light[3]
PhongShader MVP 變換，輸出世界空間法向量 插值法向量，逐像素計算 N·L Vec3f normal[3]
TextureShader MVP 變換，輸出 UV 採樣貼圖，Lambert 光照 Vec2f uv[3]
Ch25 重點整理：
  • IShader 只有兩個純虛函數：vertex(face, vert) 和 fragment(bar, color)
  • Uniform = 子類別成員變數，DrawCall 前設定，vertex 和 fragment 都可讀
  • Varying = 子類別陣列成員（長度 3），vertex() 寫入，fragment() 讀取（插值後）
  • fragment() 回傳 false = discard（對應 GLSL 的 discard）
  • 所有 Shader 子類別共用同一個光柵化迴圈——光柵化器只知道 IShader&，不知道子類別
  • 開發新效果：繼承 IShader，只寫 vertex() 和 fragment() 即可

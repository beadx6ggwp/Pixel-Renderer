Bresenham 直線演算法 — d₁ d₂ 距離比較法

按照原始論文的推導脈絡：定義決策參數 P，用增量更新取代重複計算

1
問題設定（限制條件：0 ≤ m ≤ 1）

先處理斜率在 0 到 1 之間的情況（dx ≥ dy ≥ 0，往右往上）。

直線方程：y = mx + b，其中 m = dy/dx。

因為 0 ≤ m ≤ 1，所以：

  • 每步 x 必走 +1（x 是主軸）
  • y 要嘛不動，要嘛 +1
在 x = xi+1 這個位置，理想直線的 y 值 = m(xi+1) + b， 落在 Yi 和 Yi+1 之間（因為 m ≤ 1）。
問題變成：選 Yi 還是 Yi+1 = Yi+1？
2
定義 d₁、d₂：理想線到兩個候選像素的距離

設 xi+1 位置的理想 y 值 = m(xi+1) + b，寫出 d₁ 和 d₂：

$\begin{array}{ll} d_1 = [m(x_{i+1}) + b] - Y_i & \textcolor{#6c7086}{\text{// 下方候選像素到理想 y 的距離}} \\[8pt] d_2 = Y_{i+1} - [m(x_{i+1}) + b] & \\ \quad = (Y_i + 1) - [m(x_{i+1}) + b] & \textcolor{#6c7086}{\text{// } Y_{i+1} = Y_i + 1} \end{array}$
決策規則很直接：
d₁ < d₂ → 理想線靠近下方 → 選 Yᵢ（不升 y）
d₁ > d₂ → 理想線靠近上方 → 選 Yᵢ₊₁（升 y）
等價於判斷 d₁ − d₂ 的正負。
3
計算 d₁ − d₂（還有浮點數）
$\begin{array}{rl} d_1 - d_2 &= [m(x_{i+1}) + b - Y_i] - [(Y_i+1) - m(x_{i+1}) - b] \\[4pt] &= 2m(x_{i+1}) + 2b - 2Y_i - 1 \\[4pt] &\textcolor{#6c7086}{(\text{代入 } x_{i+1} = x_i + 1)} \\[4pt] &= 2mx_i + 2m + 2b - 2Y_i - 1 \end{array}$

代入 m = dy/dx：

$d_1 - d_2 = 2\cdot\frac{\Delta y}{\Delta x}\cdot x_i + 2\cdot\frac{\Delta y}{\Delta x} + 2b - 2Y_i - 1$

兩側 × Δx（Δx > 0，不改變正負）：

$\textcolor{#f9e2af}{\Delta x(d_1 - d_2) = 2\Delta y \cdot x_i - 2\Delta x \cdot Y_i + 2\Delta y + \Delta x(2b-1)} \quad \text{← ④式}$

現在右側全是整數（dx, dy, xᵢ, Yᵢ 都是整數；b 還有問題，後面處理）。

4
定義決策參數 Pᵢ

把 ④式右側拆成「隨 i 變動的部分」和「常數 C」：

$C = 2\Delta y + \Delta x(2b-1) \qquad \textcolor{#6c7086}{\text{（常數，整個迴圈不變）}}$ $\textcolor{#f9e2af}{P_i = \Delta x(d_1 - d_2) = 2\Delta y \cdot x_i - 2\Delta x \cdot Y_i + C}$
決策規則換成：
Pi < 0 → d₁ < d₂ → 選 Yi（不升 y）
Pi > 0 → d₁ > d₂ → 選 Yi+1（升 y）
5
推導增量公式（核心！）

Pᵢ 的計算需要 xᵢ 和 Yᵢ，每步重算太浪費。 用 Pᵢ₊₁ − Pᵢ 推出增量，只做一次加法。

$\begin{array}{rl} P_{i+1} &= 2\Delta y \cdot x_{i+1} - 2\Delta x \cdot Y_{i+1} + C \\[6pt] P_{i+1} - P_i &= 2\Delta y \cdot [x_{i+1} - x_i] - 2\Delta x \cdot [Y_{i+1} - Y_i] \\[4pt] &\textcolor{#6c7086}{(x_{i+1} - x_i = 1 \text{，每步 x 必走})} \\[6pt] \textcolor{#f9e2af}{P_{i+1} - P_i} &\textcolor{#f9e2af}{= 2\Delta y - 2\Delta x \cdot (Y_{i+1} - Y_i)} \end{array}$

Yᵢ₊₁ − Yᵢ 就是「y 有沒有升格」，只有兩種值：

Case 1：Pᵢ < 0，選 Yᵢ（y 不動）
Yᵢ₊₁ = Yᵢ，所以 Yᵢ₊₁ − Yᵢ = 0

$\textcolor{#f38ba8}{P_{i+1} = P_i + 2\Delta y}$

Case 2：Pᵢ > 0，選 Yᵢ₊₁（y 升一格）
Yᵢ₊₁ = Yᵢ + 1，所以 Yᵢ₊₁ − Yᵢ = 1

$\textcolor{#a6e3a1}{P_{i+1} = P_i + 2\Delta y - 2\Delta x}$
每步只需要一次加法。沒有乘法，沒有浮點數，沒有除法。
6
計算初始值 P₀

起點 (x₀, y₀) 在直線上，所以 b = y₀ − m·x₀ = y₀ − (dy/dx)·x₀。

代入 P₀（i = 0 時的值，在第一步畫完 (x₀, y₀) 後，用來判斷下一步）：

$\begin{array}{rl} P_0 &= 2\Delta y \cdot x_0 - 2\Delta x \cdot y_0 + C \\[4pt] &\textcolor{#6c7086}{(\text{展開 } C = 2\Delta y + \Delta x(2b-1))} \\[4pt] &= 2\Delta y \cdot x_0 - 2\Delta x \cdot y_0 + 2\Delta y + 2\Delta x \cdot b - \Delta x \\[4pt] &\textcolor{#6c7086}{(\text{代入 } b = y_0 - \tfrac{\Delta y}{\Delta x} x_0 \;\Rightarrow\; 2\Delta x \cdot b = 2\Delta x \cdot y_0 - 2\Delta y \cdot x_0)} \\[4pt] &= 2\Delta y \cdot x_0 - 2\Delta x \cdot y_0 + 2\Delta y + (2\Delta x \cdot y_0 - 2\Delta y \cdot x_0) - \Delta x \\[6pt] \textcolor{#f9e2af}{P_0} &\textcolor{#f9e2af}{= 2\Delta y - \Delta x} \end{array}$
P₀ = 2dy − dx
起點完全消掉了！初始值只取決於 dx 和 dy，跟起點座標無關。 這是因為我們從 (x₀, y₀) 這個線上的點出發，b 的部分剛好被消掉。
7
完整演算法（slope 0~1 版）
// 初始化
P = 2Δy − Δx // P₀

loop（x 從 x₀ 走到 x₁）：
  畫 (x, y)
  x += 1 // x 每步必走

  if P < 0: // d₁ < d₂，靠近下方，y 不動
    P += 2Δy
  else: // d₁ > d₂，靠近上方，y 升一格
    y += 1
    P += 2Δy − 2Δx
8
跑一遍數字驗證：(0,0) → (7,3)

dx=7, dy=3，P₀ = 2×3 − 7 = −1

步畫P P < 0？動作新 P
0(0,0)−1 是y 不動 −1 + 6 = 5
1(1,0)5 否y++ → y=1 5 + 6 − 14 = −3
2(2,1)−3 是y 不動 −3 + 6 = 3
3(3,1)3 否y++ → y=2 3 + 6 − 14 = −5
4(4,2)−5 是y 不動 −5 + 6 = 1
5(5,2)1 否y++ → y=3 1 + 6 − 14 = −7
6(6,3)−7 是y 不動 —
7(7,3)抵達終點 ✓

結果：(0,0)(1,0)(2,1)(3,1)(4,2)(5,2)(6,3)(7,3) — 正確。

9
對應到程式碼的版本

程式碼用的是通用版（任意斜率、任意方向），把上面的 P 做了一個等價變換：

// 程式碼的 err 和 P 的關係
err = −P / 2 + (Δx − Δy) / 2 // 等比例縮放 + 移位
// 等價，只是換個基準點讓兩軸對稱

// 程式碼的初始值
err = Δx − Δy // 對應 P₀ = 2Δy − Δx（符號相反）

// 程式碼的判斷
e2 = 2 × err
if e2 > −Δy → x 走 // 對稱地處理 x 軸
if e2 < Δx → y 走 // 對應 P < 0 的條件
程式碼把「x 方向」和「y 方向」都做了同樣的 d₁/d₂ 分析， 合併成兩個獨立的 if，讓單一迴圈同時處理任意斜率。 兩個 if 可以同時觸發（斜率接近 1 的步驟，x 和 y 都走）。

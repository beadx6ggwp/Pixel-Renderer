2
包圍盒計算

三角形的包圍盒（Axis-Aligned Bounding Box, AABB）是最小的軸對齊矩形， 它能完全包含三角形。算法很直接：

xmin = floor( min(V0.x, V1.x, V2.x) ) xmax = ceil( max(V0.x, V1.x, V2.x) ) ymin = floor( min(V0.y, V1.y, V2.y) ) ymax = ceil( max(V0.y, V1.y, V2.y) ) Clamp 到畫面邊界： xmin = max(xmin, 0) xmax = min(xmax, width-1) ymin = max(ymin, 0) ymax = min(ymax, height-1)
為什麼要用整數？
頂點座標通常是浮點數（投影後的結果）。 floor/ceil 把它轉成整數像素座標，確保不會漏掉邊界上的像素。 xmax 需要 ceil 而非 floor，避免截掉最右邊那列。

3
為什麼用包圍盒，而不是掃描線法？

傳統的掃描線法（Scanline）把三角形分成梯形，逐列找出左右邊界再填色。 這曾是最快的做法，因為它避免了對三角形外的像素做測試。 但現代 GPU 和 CPU 都更適合包圍盒法：

面向 掃描線法（Scanline） 包圍盒法（Bounding Box）
邏輯複雜度 高：要找左右交點，處理各種頂點排列 低：兩層迴圈 + 三次叉積
程式碼量 多：mini3d.c 的梯形分割需要 ~100 行 少：核心迴圈約 20 行
平行化 難：掃描線之間有依賴 易：每個像素獨立測試，GPU 並行
SIMD 友善 不規則的邊界計算難向量化 規則的矩形迴圈，SIMD 4/8 像素同時算
浪費的測試 幾乎沒有（只掃描三角形覆蓋的行） 有（包圍盒四角的區域），但 Edge Test 很快
現代 GPU 使用 不使用 標準做法（2×2 Quad 批次處理）
一句話總結： 掃描線法用更少的測試換取更複雜的邊界計算；包圍盒法用更多的（但極快的）測試換取極簡的程式碼。 在現代硬體上，Edge Function 測試便宜到讓包圍盒法完勝。

4
完整演算法

void DrawTriangle(Vec2 V0, Vec2 V1, Vec2 V2,
                 Color C0, Color C1, Color C2) {

    // ── Step 1：有向面積 & 退化檢查 ──────────────────────
    float area = (V1.x-V0.x)*(V2.y-V0.y) - (V1.y-V0.y)*(V2.x-V0.x);
    if (area == 0) return; // 退化三角形（共線），面積為 0
    float inv_area = 1.0f / area;

    // ── Step 2：包圍盒 ────────────────────────────────────
    int xmin = max(0, (int)floor(min({V0.x, V1.x, V2.x})));
    int xmax = min(width-1, (int)ceil( max({V0.x, V1.x, V2.x})));
    int ymin = max(0, (int)floor(min({V0.y, V1.y, V2.y})));
    int ymax = min(height-1,(int)ceil( max({V0.y, V1.y, V2.y})));

    // ── Step 3：Edge Function 係數 ────────────────────────
    // E0(x,y) = a0*x + b0*y + c0 (對應邊 V0→V1，判斷 V2 所在側)
    float a0=V1.y-V0.y, b0=V0.x-V1.x, c0=V1.x*V0.y-V0.x*V1.y;
    float a1=V2.y-V1.y, b1=V1.x-V2.x, c1=V2.x*V1.y-V1.x*V2.y;
    float a2=V0.y-V2.y, b2=V2.x-V0.x, c2=V0.x*V2.y-V2.x*V0.y;

    // ── Step 4：掃描迴圈 ──────────────────────────────────
    for (int y = ymin; y <= ymax; y++) {
        float e0 = a0*xmin + b0*y + c0;
        float e1 = a1*xmin + b1*y + c1;
        float e2 = a2*xmin + b2*y + c2;

        for (int x = xmin; x <= xmax; x++) {
            // Edge Function 內外測試
            if ((e0>=0&&e1>=0&&e2>=0)||(e0<=0&&e1<=0&&e2<=0)) {

                // 重心座標（Ch10）
                float w0 = e0 * inv_area;
                float w1 = e1 * inv_area;
                float w2 = e2 * inv_area;

                // 插值顏色
                Color c = w0*C0 + w1*C1 + w2*C2;
                SetPixel(x, y, c);
            }
            // 增量：移到下一列（x+1）
            e0 += a0; e1 += a1; e2 += a2;
        }
    }
}

黃色高亮的三行是每列的「起始值計算」，之後靠 e += a 增量更新。 整個掃描過程，每個像素只做三次加法和三次比較——這就是效能的來源。

5
邊界情況處理

退化三角形（面積 = 0）

三個頂點共線時，有向面積為 0，inv_area 會是無限大。 在 Step 1 直接 return 處理，什麼都不畫——這是正確行為。

浮點數比較陷阱：
area == 0 對整數座標可行，但浮點座標建議用 fabsf(area) < 1e-6f， 避免投影後極細的三角形因精度問題被錯誤跳過。

三角形完全在畫面外

clamp 後若 xmin > xmax 或 ymin > ymax，包圍盒為空，迴圈直接不執行。 不需要額外的邊界檢查。

三角形部分超出畫面

clamp 到 [0, width-1] 之後，掃描範圍自動縮小到畫面內。 Edge Function 仍然正確（係數 a/b/c 不受 clamp 影響）， 只是有些「本來在三角形內」的像素被 clamp 截掉了——這正是我們想要的行為。

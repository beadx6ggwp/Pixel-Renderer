Bresenham — 8 個方向完整分析

從已推導的「左下→右上緩坡」出發，系統性地推廣到所有方向

1
全部 8 個方向（8 個 Octant）

把座標系分成 8 個扇形，每個扇形有不同的主軸和方向：

觀察到兩個關鍵對稱性：

  1. 方向對稱：①②和⑤⑥是鏡像（左右翻），④③和⑧⑦是鏡像（上下翻）
    → 只要控制 sx, sy 的正負，誤差計算完全一樣
  2. 軸對稱：①和②是鏡像（沿 45° 翻轉），緩坡換成陡坡
    → 只要交換 dx 和 dy 的角色（把 y 當主軸）
所以不需要寫 8 套程式碼。用 3 個參數 統一處理：
sx（x 方向）、sy（y 方向）、dx vs dy（誰是主軸）
2
第一個對稱：方向（sx, sy）

已知 ① 緩坡右上（dx≥dy, sx=+1, sy=+1）的完整推導。 現在看方向翻轉對 d₁, d₂ 有什麼影響。

Case ⑧：緩坡右下（sx=+1, sy=−1）

線從 (x₀,y₀) 往右下走，斜率為負，|m| ≤ 1。

下一個候選像素是 (x+1, Yᵢ) 或 (x+1, Yᵢ−1)（y 可能下降）。

d₁, d₂ 定義跟著翻轉：

// 緩坡右下（sy = −1） d₁ = Yᵢ − [m(xᵢ₊₁) + b] // 理想 y 到上方候選（不動那個） d₂ = [m(xᵢ₊₁) + b] − Yᵢ₋₁ // 理想 y 到下方候選（下降那個）

展開 d₁ − d₂，過程完全一樣（只是 Yᵢ₊₁ 換成 Yᵢ₋₁ = Yᵢ−1）：

d₁ − d₂ = 2Yᵢ − 2m(xᵢ₊₁) − 2b − 1 // × dx dx(d₁−d₂) = −2dy·xᵢ + 2dx·Yᵢ + (常數)
結論：d₁−d₂ 的符號和 ① 完全一樣，只是 dy 的符號翻轉了。
但我們已經用 dy = |y₂−y₁|（取絕對值），再用 sy = −1 控制 y 移動方向。
所以 P 的計算和判斷條件完全不變，只需要在 y 步進時用 y += sy。

同理，⑤ 緩坡左下（sx=−1, sy=−1）、④ 緩坡左上（sx=−1, sy=+1）等等， 全都是相同的推導，只是 x += sx 和 y += sy。

方向對稱的結論：
取 dx = |x₂−x₁|，dy = |y₂−y₁|（全正），
sx = sign(x₂−x₁)，sy = sign(y₂−y₁)（記方向），
誤差 P 的公式和判斷條件對 ①②③④⑤⑥⑦⑧ 全部一樣。
3
第二個對稱：陡坡（交換主軸）

當 dy > dx（斜率 |m| > 1），x 每走一步 y 可能要走好幾步。 這時候 x 不是主軸了，y 才是。

① 緩坡右上（dx ≥ dy）

x 每步必走 +1 y 在 Yᵢ 或 Yᵢ+1 選一個 P₀ = 2dy − dx P < 0: P += 2dy y 不動 P ≥ 0: y++; P += 2dy−2dx y 升

② 陡坡右上（dy > dx）

y 每步必走 +1 x 在 Xᵢ 或 Xᵢ+1 選一個 P₀ = 2dx − dy P < 0: P += 2dx x 不動 P ≥ 0: x++; P += 2dx−2dy x 走

推導過程完全對稱，只是把 dx 和 dy 對調，x 和 y 對調。

再推一遍陡坡的 d₁, d₂（以 ② 陡坡右上為例）：

// y 是主軸，每步 y+1；x 在 Xᵢ 或 Xᵢ+1 選一個 // 直線方程改用 x = (1/m)·y + b'，斜率換成 dx/dy d₁ = [x_理想 at yᵢ₊₁] − Xᵢ // 理想 x 到左方候選（不動）的距離 d₂ = Xᵢ₊₁ − [x_理想 at yᵢ₊₁] // 右方候選（前進）到理想 x 的距離 // 展開 d₁−d₂，兩側 × dy： P₀ = 2dx − dy P < 0: x 不動, P += 2dx P ≥ 0: x++, P += 2dx − 2dy
陡坡和緩坡的公式結構完全相同，只是 dx ↔ dy 互換。
這也說明為什麼程式碼裡的 err = dx−dy 對兩種情況都成立—— dx−dy 的符號本身就反映了主軸是誰。

5
程式碼如何把 8 個 Case 合併成一個迴圈

程式碼用 err = dx − dy 作為初始值，同時做 x 和 y 的判斷， 讓緩坡和陡坡自動選對主軸。

關鍵是把兩個判斷做成對稱的兩個獨立 if（不是 if-else）：

err = dx − dy // 注意：這等於 −P₀（對比 ① 的 P₀ = 2dy−dx，除以 2 再取負） e2 = 2 × err // 快照，相當於 −P // 對稱判斷一：要不要走 x？ if e2 > −dy: x += sx; err −= dy // 對稱判斷二：要不要走 y？ if e2 < dx: y += sy; err += dx

把這兩個判斷和上面 8 個 case 的 P 對應起來：

情況e2 和 −dy, dx 的關係效果對應 P 判斷
緩坡（dx>dy），e2 大 e2 > −dy（幾乎總是成立）
e2 < dx（不一定）		 x 必走；y 偶爾走 x 是主軸 → y 的判斷對應 P
陡坡（dy>dx），e2 小 e2 > −dy（不一定）
e2 < dx（幾乎總是成立）		 y 必走；x 偶爾走 y 是主軸 → x 的判斷對應 P
斜率 = 1（dx=dy） 兩個條件都成立 x 和 y 都走（對角線） 兩個 if 同時觸發
兩個獨立 if（不是 if-else）正是為了處理斜率 = 1 的情況，讓 x 和 y 同時步進。
用 e2 的快照而不是即時的 err，是因為第一個 if 改了 err 之後， 不能讓它影響第二個 if 的判斷。

總結
從 ① 推廣到全部 8 個方向，只需要三個步驟：

1. 方向：sx = sign(dx_raw)，sy = sign(dy_raw)，dx dy 取絕對值
      → 步進時用 x += sx，y += sy

2. 主軸：dx ≥ dy 時 x 是主軸，dy > dx 時 y 是主軸
      → 程式碼用 err = dx − dy + 兩個對稱 if，自動選對主軸

3. 同時步進：兩個 if 獨立（不是 if-else），處理斜率 = 1 的對角走法
      → 用 e2 快照，避免第一個 if 改 err 後影響第二個判斷

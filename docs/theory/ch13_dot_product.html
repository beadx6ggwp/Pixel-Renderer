PART 4 · CHAPTER 13
點積（Dot Product）
從「兩向量的乘法」到「光照角度計算」，N·L 是所有光照模型的基礎
dot product 投影 cosθ N·L 光照 正規化

Part 3 結束後，你已經能在螢幕上正確填滿三角形、做深度測試。 但那些三角形的頂點是直接給的螢幕座標——現實中，3D 物件存在於世界空間， 需要一連串的矩陣變換才能投影到螢幕。

Part 4 的目標是建立這套變換的數學基礎。一切從最基本的 向量運算 開始。 點積是其中最常用的工具，幾乎每一個光照計算都離不開它。

1
定義：兩種等價的寫法

點積（Dot Product，又叫內積 Inner Product）接受兩個同維度的向量，回傳一個純量（scalar）。 它有兩種完全等價的計算方式：

代數定義（直接計算）
$\mathbf{a} \cdot \mathbf{b} = a_x b_x + a_y b_y + a_z b_z$

\(\mathbf{a} = (a_x, a_y, a_z)\)，\(\mathbf{b} = (b_x, b_y, b_z)\)

幾何定義（夾角）
$\mathbf{a} \cdot \mathbf{b} = |\mathbf{a}|\,|\mathbf{b}|\cos\theta$

\(|\mathbf{a}| = \sqrt{a_x^2+a_y^2+a_z^2}\)，θ 為 a 與 b 的夾角

兩個定義描述同一件事。代數定義方便計算；幾何定義揭示意義。

具體例子

$\mathbf{a} = (1,\;2,\;3),\quad \mathbf{b} = (4,\;{-1},\;0)$ $\mathbf{a} \cdot \mathbf{b} = 1{\times}4 + 2{\times}({-1}) + 3{\times}0 = 4 - 2 + 0 = \textcolor{#a6e3a1}{2}$
點積是純量，沒有方向。
這和叉積（下一章）不同。叉積的結果是向量；點積只是一個數字。

2
幾何意義：投影

從幾何定義 a·b = |a||b|cosθ 可以推導出點積最直觀的意義： a 在 b 方向上的投影長度，乘以 |b|。

「有號」的意思是：cosθ 的正負告訴你方向。這讓點積能直接回答 「這兩個向量是同一個方向還是反方向？」

$\begin{array}{rcl} \textcolor{#a6e3a1}{\mathbf{a} \cdot \mathbf{b} > 0} & \Rightarrow & \theta < 90°\text{，同側（夾角銳角）}\\[6pt] \textcolor{#cba6f7}{\mathbf{a} \cdot \mathbf{b} = 0} & \Rightarrow & \theta = 90°\text{，垂直（正交）}\\[6pt] \textcolor{#f38ba8}{\mathbf{a} \cdot \mathbf{b} < 0} & \Rightarrow & \theta > 90°\text{，反側（夾角鈍角）} \end{array}$

3
常用恆等式

自點積 = 長度的平方

$\mathbf{a} \cdot \mathbf{a} = a_x^2 + a_y^2 + a_z^2 = |\mathbf{a}|^2 \implies |\mathbf{a}| = \sqrt{\mathbf{a} \cdot \mathbf{a}}$

這個恆等式很有用：需要向量長度時，先算點積再開根號， 比分別平方再相加略快（程式碼層面一樣，但語意更清晰）。

正規化（Normalization）

長度為 1 的向量叫單位向量（unit vector），通常寫作 \(\hat{a}\) 或 \(\hat{n}\)。 把任意向量除以自身長度就能得到：

$\hat{\mathbf{a}} = \dfrac{\mathbf{a}}{|\mathbf{a}|} = \dfrac{\mathbf{a}}{\sqrt{\mathbf{a} \cdot \mathbf{a}}} \implies \hat{\mathbf{a}} \cdot \hat{\mathbf{a}} = 1$
光照計算幾乎都在用單位向量。
法向量 N、光照方向 L、視角方向 V——這三個向量在 Phong 模型中全部需要正規化， 因為我們只關心「方向」，不關心「長度」。 傳入 GPU shader 前通常已經正規化。

交換律與分配律

$\begin{array}{rll} \mathbf{a} \cdot \mathbf{b} &= \mathbf{b} \cdot \mathbf{a} & \text{（交換律）}\\[6pt] \mathbf{a} \cdot (\mathbf{b} + \mathbf{c}) &= \mathbf{a}\cdot\mathbf{b} + \mathbf{a}\cdot\mathbf{c} & \text{（分配律）}\\[6pt] (k\mathbf{a}) \cdot \mathbf{b} &= k(\mathbf{a} \cdot \mathbf{b}) & \text{（純量結合律）} \end{array}$

4
核心應用：N·L 光照

點積在圖形學最重要的應用就是 Lambert（朗伯）漫反射光照模型：

$\text{diffuse} = \max\!\left(0,\;\textcolor{#cba6f7}{\hat{N}} \cdot \textcolor{#f9e2af}{\hat{L}}\right) \times \text{light\_color} \times \text{surface\_color}$
\(\textcolor{#cba6f7}{\hat{N}}\) = 表面法向量（單位向量，垂直於三角形平面）
\(\textcolor{#f9e2af}{\hat{L}}\) = 指向光源的方向向量（單位向量）
\(\hat{N}\cdot\hat{L} = \cos\theta\)，θ 是光線照射角度

max(0, N·L) 裡的 max(0, …) 是截斷負值：背光面的點積是負數， 但我們不希望光照強度是負的（那沒有物理意義），所以截斷為 0。

這是 Phong 光照模型中「漫反射」那一項的完整計算。
之後引入法向量（Ch14 3D 叉積計算）和 MVP 矩陣（Ch16）後， 我們的 DrawTriangle 就能根據光照角度為每個像素著色——而非固定顏色。

5
其他應用

高光計算：V·R（Phong 反射）

Phong 模型的高光項需要計算「視角方向 V 和反射光方向 R 的夾角」：

$\text{specular} = \max\!\left(0,\;\hat{V} \cdot \hat{R}\right)^{\text{shininess}}$

\(\hat{R} = 2(\hat{N}\cdot\hat{L})\hat{N} - \hat{L}\)（反射向量，由 N 和 L 推導）；shininess 越大，高光越小越亮（金屬感）

方向測試：背面判斷的推廣

Ch12 的背面剔除用的是 2D 叉積的符號。在 3D 空間中，也可以用點積判斷： 如果三角形法向量 N 和攝影機方向 V 的點積 < 0，代表攝影機在三角形背面。

$\begin{array}{ll} \textcolor{#a6e3a1}{\hat{N} \cdot \hat{V} > 0} & \Rightarrow\;\text{正面（front-face），需要渲染}\\[6pt] \textcolor{#f38ba8}{\hat{N} \cdot \hat{V} < 0} & \Rightarrow\;\text{背面（back-face），可以剔除} \end{array}$

Ch12 的 2D 叉積法是這個的螢幕空間版本。

距離與投影

用途 公式 說明
向量長度 |a| = √(a·a) 自點積開根號
點到平面距離 d = (P − Q) · N̂ P 到過 Q 點法向量 N̂ 平面的距離
向量投影 proj = (a·b̂) b̂ a 在 b̂ 方向的投影向量
正交分解 a_perp = a − (a·b̂)b̂ a 垂直於 b 的分量（Gram-Schmidt）

6
C++ 實作

點積的實作極簡單。這是之後 Vec3 類別的核心運算：

struct Vec3 { float x, y, z; // 長度平方（避免開根號，適合比較距離） float length_sq() const { return x*x + y*y + z*z; } // 長度 float length() const { return std::sqrt(length_sq()); } // 正規化（回傳單位向量） Vec3 normalize() const { float l = length(); return {x/l, y/l, z/l}; } }; // 點積 float dot(Vec3 a, Vec3 b) { return a.x*b.x + a.y*b.y + a.z*b.z; } // 漫反射光照計算 float diffuse = std::max(0.0f, dot(N.normalize(), L.normalize()));
注意：呼叫 dot(N, L) 前一定要確認 N 和 L 都已正規化， 否則結果不是 cosθ，而是 |N||L|cosθ——計算光照強度時會出錯。 養成習慣：傳入 normalize() 後的結果，或在函式內部正規化。

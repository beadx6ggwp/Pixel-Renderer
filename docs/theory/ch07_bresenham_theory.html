兩種推導視角與背後理論

誤差累積法 vs d₁d₂ 距離法 — 它們是同一件事嗎？哪裡不同？能推廣到哪裡？

1
兩種方法的哲學差異

兩種方法產生完全相同的像素序列，但出發點不同：

▌ 誤差累積法

問的問題：

「我累積欠了多少 y？欠得夠多就還一格。」

  • 視角：動態，追蹤狀態
  • 思考單位：時間序列，累積量
  • 發現它需要：程式設計直覺
  • 推廣性：直線專用
▌ d₁d₂ 距離法

問的問題：

「兩個候選像素，哪個離理想線更近？」

  • 視角：靜態，比較兩個選項
  • 思考單位：空間距離，幾何關係
  • 發現它需要：幾何 + 計算幾何概念
  • 推廣性：可推廣到任何隱式曲線
類比：你要過河。
誤差累積法像是：「我走了幾步了，偏移了多少，要不要修正？」
d₁d₂ 法像是：「左腳踩這塊石頭近，還是右腳踩那塊近？」
兩個判斷等價，但思考方式完全不同。

2
數學上它們是同一件事

用具體數字說明。(0,0) → (7,3)，dx=7，dy=3。

誤差累積法的 err

初始：err = 0 每步 x：err += 2dy = 6 當 err ≥ dx（= 7）：y++, err -= 2dx = 14

d₁d₂ 法的 P

初始：P₀ = 2dy − dx = 6 − 7 = −1 P < 0：P += 2dy = 6 P ≥ 0：y++, P += 2dy − 2dx = 6 − 14 = −8

兩者對照

// 兩者的精確關係 P = err − dx // 等價性： err ≥ dx ⟺ err − dx ≥ 0 ⟺ P ≥ 0 （升 y 的條件） err < dx ⟺ P < 0 （不升 y 的條件）
兩個方法追蹤的是同一個量，只差一個常數 dx 的位移。
判斷「err 是否 ≥ dx」等價於判斷「P 是否 ≥ 0」。
任何時刻，兩個方法做出完全相同的決定。

3
中點測試（Midpoint Test）—— d₁d₂ 的真正來源

d₁d₂ 法不是為直線特別設計的。它的核心是一個更通用的技巧：

中點測試（Midpoint Test）
有一條曲線 F(x, y) = 0。F > 0 在曲線一側，F < 0 在另一側。
面對兩個候選格點 A 和 B，取它們的中點 M。
計算 F(M) 的符號 → 告訴你曲線偏向哪側 → 選那一側的格點。

直線的中點測試

直線：F(x, y) = dy·x − dx·y + C = 0

候選像素：(x+1, Yᵢ) 和 (x+1, Yᵢ+1) 中點 M = (x+1, Yᵢ + 0.5) F(M) = dy·(x+1) − dx·(Yᵢ+0.5) + C F(M) > 0 → 曲線在中點下方 → 選上方像素 Yᵢ+1 F(M) < 0 → 曲線在中點上方 → 選下方像素 Yᵢ // F(M) × 2（消掉 0.5）= d₁d₂ 法的 P！

圓的中點測試（推廣）

圓：F(x, y) = x² + y² − r² = 0

// 畫圓的第一象限，y 從 r 往下，x 從 0 往右 候選像素：E = (x+1, y) 或 SE = (x+1, y-1) 中點 M = (x+1, y − 0.5) F(M) = (x+1)² + (y−0.5)² − r² F(M) > 0 → M 在圓外 → 選 SE（往內走） F(M) ≤ 0 → M 在圓內 → 選 E（保持高度） // 增量更新（同樣的技巧）： P₀ = 1 − r P ≤ 0: P += 2x + 3 （選 E） P > 0: P += 2x − 2y + 5; y−− （選 SE）
這就是為什麼 d₁d₂ 法重要：
把「選哪個像素」這個問題，統一成「對某個 F(x,y) 在中點求符號」。
只要能寫出曲線的隱式方程，套用同樣的框架，就能推導出它的 Midpoint 演算法。
這個框架推廣到了：直線 → 圓 → 橢圓 → 貝茲曲線 → 任何代數曲線。

4
跟數值分析的關係

關係一：符號判斷（Bisection Method 的本質）

數值分析找根的二分法：

// f(a) 和 f(b) 符號相反 → 根在 [a,b] 之間 f(a) · f(b) < 0 → 根在 [a, b] 取中點 m = (a+b)/2，判斷 f(m) 符號，縮小區間

Bresenham 的中點測試：

// 兩個候選格點之間的中點 M F(M) > 0 → 曲線在 M 的這一側 → 選那一側的格點

同樣的概念：用函數在中間點的符號，判斷目標在哪一側。 二分法在連續區間上做，Bresenham 在離散格點上做。

關係二：DDA 和有限差分（Finite Difference）

Bresenham 的前身是 DDA（Digital Differential Analyzer）， 1950 年代的硬體，直接計算微分方程：

// DDA：把微分方程數值積分 // 直線 dy/dx = m 的數值解： y(x + Δx) = y(x) + m · Δx // 在像素上：Δx = 1，所以 y_next = y + m // 浮點加法，每步一次

這就是數值分析的 Euler 方法：用差分近似微分。

DDA 的問題是每步都做浮點運算，在 1960 年代很昂貴。

Bresenham（1965）的貢獻：把 DDA 的浮點累積替換成整數決策， 本質上是把「連續的 y 值追蹤」轉成「離散的格點選擇」：

DDA：追蹤 y（連續） → Bresenham：追蹤 err（離散決策） y += m → err += dy（整數），判斷是否 ≥ dx

關係三：增量計算（Incremental Computation）

兩種方法共用的核心優化技巧：

// 笨方法：每步重算 P_i = 2dy·xᵢ − 2dx·Yᵢ + C // 需要乘法 // 聰明方法：只算增量 P_{i+1} − P_i = 2dy − 2dx·(Yᵢ₊₁ − Yᵢ) P_{i+1} = P_i + 常數 // 只需要加法

這個技巧在數值分析叫 差分法（Method of Differences）， 17 世紀 Newton、Gregory 就在用：計算多項式值時，用逐階差分取代重新代入。

// Newton Forward Difference 計算 f(x+h) f(x+h) = f(x) + Δf(x) + Δ²f(x)/2! + ... // Bresenham 的增量更新就是一階差分 P_{i+1} = P_i + ΔP （ΔP 是常數，因為 P 是 x 的一次函數）

關係四：格點近似（Lattice Point Approximation）

「哪個整數格點最接近連續曲線」這個問題屬於 數論幾何（Geometry of Numbers） 和 Diophantine 近似 的領域。

直線的格點問題相對簡單；推廣到代數曲線（圓、橢圓）就是計算幾何的研究對象。 Bresenham 把它做成了 O(1) 每像素的演算法。

5
歷史脈絡

1940s
DDA 硬體
Digital Differential Analyzer。類比/數位混合硬體，用 Euler 方法積分微分方程。直線就是解 dy/dx = m，每步 y += m（浮點）。
1965
Bresenham 直線演算法
J. E. Bresenham 在 IBM 發表。把 DDA 的浮點累積整數化，用誤差累積法。論文的推導是誤差累積視角，非常工程導向。
1967
Pitteway 的推廣
M. L. V. Pitteway 重新推導，用幾何距離視角，並推廣到圓錐曲線（圓、橢圓、拋物線）。這是 d₁d₂ / 中點法的前身。
1977
Bresenham 圓演算法
Bresenham 本人也發表了圓的版本。同樣用整數誤差，但被認為比 Midpoint 版略難理解。
1985
Midpoint Circle / Ellipse Algorithm
Van Aken、Foley 等人在計算機圖形學教科書（FVFH）中系統化 Midpoint 方法，統一了直線和曲線的推導框架。這才是現代教科書的標準版本。
有趣的歷史細節：
Bresenham 1965 年論文的推導其實很接近誤差累積法，不是教科書現在教的 d₁d₂ 版。 d₁d₂ / 中點法是後人重新詮釋和推廣的結果。 所以你筆記裡的推導方式其實是「現代版」，比原始論文更有系統。

6
要能「自己發現」d₁d₂ 需要的知識地圖

必
隱式曲線 F(x,y) = 0
知道任何曲線都可以寫成「F(x,y) 等於零的點的集合」。
直線：F = dy·x − dx·y + C。圓：F = x² + y² − r²。
更重要的是：F > 0 和 F < 0 代表在曲線的不同側。
來源：解析幾何、微積分的隱函數定理。
必
中點測試的模式
「面對兩個選項 A 和 B，在它們中間代入 F，判斷曲線偏向哪側，就選那側。」
這個模式不是直覺能想到的，需要見過至少一次（才能知道這是個可用的工具）。
來源：計算幾何、Foley 等人的 Computer Graphics 教科書。
加
增量計算的意識
知道「如果某個量每步只差一個常數，就不用重算，只做加法」。
這是一般的演算法設計直覺，不需要特別的數學背景。
來源：任何演算法入門課都會建立這個直覺。
加
整數代替浮點的技巧
知道「乘以分母可以消掉除法，不影響正負號的比較」。
這是很基本的代數操作，但需要有意識地去用。
來源：基礎代數。
選
數值分析（非必要，但有幫助）
知道 DDA / Euler 方法，能幫你理解「Bresenham 在改進什麼」。
知道差分法（Newton Forward Difference），能幫你理解增量計算的數學基礎。
知道二分法，能幫你理解中點測試的更深連結。
這些是「理解更深」的背景，不是「發現演算法」的必要條件。
選
數論幾何（更深層的理論）
「連續曲線上最近的整數格點」是數論幾何的問題。
這個連結更多是事後理解，不是發現演算法的路徑。
非常 optional，屬於理論電腦科學 / 數學的範疇。

7
兩種方法各自適合的場景

場景建議用哪種原因
第一次學直線柵格化 誤差累積法 直覺，容易和程式碼對應
要推廣到圓和橢圓 d₁d₂ / 中點法 框架統一，可直接套用
要向別人解釋「為什麼這樣是對的」 d₁d₂ / 中點法 幾何意義清晰，選最近像素的理由顯然
要理解程式碼在做什麼 誤差累積法 err 的物理意義（欠了多少）直接對應迴圈狀態
要推廣到任意隱式代數曲線 d₁d₂ / 中點法 只需要換 F(x,y) 的定義，其餘框架不變
實作效能最佳化 兩者等價 最終都化成同樣的整數加法迴圈

總結

兩種方法的差異不在結果，在問題的提法。

誤差累積法問的是：「現在的狀態是什麼？下一步怎麼修正？」
d₁d₂ 法問的是：「哪個選項更接近正確答案？」

前者的語言是演算法和狀態機；後者的語言是幾何和最佳化。

理解兩種視角，才能知道「當曲線從直線換成圓的時候，改變的只有 F，其餘不變」。 這個遷移能力，才是學 d₁d₂ 法的真正收穫。

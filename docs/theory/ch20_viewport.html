PART 5 · CHAPTER 20
視口變換
NDC 的 [−1,1]³ 如何映射到螢幕像素座標——y 軸翻轉、深度重映射與 viewport_matrix() 推導
NDC 視口映射 y 軸翻轉 viewport_matrix 螢幕座標

投影矩陣（Ch18）把相機空間的頂點壓進 Clip 空間，透視除法後進入 NDC（Normalized Device Coordinates）——\(x, y, z\) 均在 \([-1, 1]\) 之間。 這是個標準化的抽象空間，與解析度無關。

最後一步：視口變換（Viewport Transform）把 NDC 對應到實際的螢幕像素座標， 這樣光柵化才知道該把三角形畫在畫面的哪個位置。

1
NDC 空間的定義

透視除法（\(\div w\)）之後，每個頂點的座標都落在：

$x_{\text{ndc}} \in [-1, 1], \quad y_{\text{ndc}} \in [-1, 1], \quad z_{\text{ndc}} \in [-1, 1]$

NDC 的座標系有以下慣例（OpenGL）：

NDC 方向含義
\(x = -1\)畫面最左
\(x = +1\)畫面最右
\(y = -1\)畫面最下（NDC y 朝上）
\(y = +1\)畫面最上（NDC y 朝上）
\(z = -1\)近平面（near）
\(z = +1\)遠平面（far）
NDC y 朝上，螢幕 y 朝下：這是最常見的混淆來源。 NDC 遵循數學座標系（y 往上增大），但大多數螢幕/圖形 API（CPU framebuffer、PNG 圖片像素） 的 y 從上往下增大——\(y = 0\) 在畫面頂端。 視口變換需要在 y 方向做翻轉。

2
視口參數與映射目標

視口（Viewport）由四個參數定義：

參數含義
\(x, y\)視口左上角在 framebuffer 中的像素座標（通常為 0, 0）
\(w\)視口寬度（像素）
\(h\)視口高度（像素）

最終目標：把 NDC 的 \([-1,1]\) 區間線性映射到像素座標範圍。

3
推導映射公式

對 \(x\) 和 \(y\) 分別做線性映射：

X 軸映射

NDC \(x \in [-1, 1]\) → 螢幕 \(x_s \in [0, W]\)（原點在左上角）：

$x_s = \frac{x_{\text{ndc}} + 1}{2} \cdot W = \frac{W}{2} \cdot x_{\text{ndc}} + \frac{W}{2}$

Y 軸映射（含翻轉）

NDC \(y \in [-1, 1]\)（\(-1\) 在下，\(+1\) 在上）→ 螢幕 \(y_s \in [0, H]\)（\(0\) 在上，\(H\) 在下）：

$y_s = \frac{1 - y_{\text{ndc}}}{2} \cdot H = -\frac{H}{2} \cdot y_{\text{ndc}} + \frac{H}{2}$

關鍵：\(1 - y_{\text{ndc}}\) 而不是 \(y_{\text{ndc}} + 1\)，這就是 y 翻轉。 NDC \(y = +1\)（頂端）→ 螢幕 \(y_s = 0\)（頂端）；NDC \(y = -1\)（底部）→ 螢幕 \(y_s = H\)（底部）。

Z 軸映射（深度重映射）

NDC \(z \in [-1, 1]\) 通常重映射到 framebuffer 的深度緩衝範圍 \([d_{\text{near}}, d_{\text{far}}]\)， 預設 \([0, 1]\)（OpenGL 的 glDepthRange）：

$z_s = \frac{d_{\text{far}} - d_{\text{near}}}{2} \cdot z_{\text{ndc}} + \frac{d_{\text{far}} + d_{\text{near}}}{2}$

對於預設的 \(d_{\text{near}} = 0, d_{\text{far}} = 1\)：

$z_s = \frac{z_{\text{ndc}} + 1}{2}$

4
視口矩陣（4×4）

把三個軸的映射整合成一個 4×4 矩陣。對於視口起點 \((x_0, y_0)\)、 寬 \(W\)、高 \(H\)、深度範圍 \([n_d, f_d]\)：

$M_{\text{viewport}} = \begin{bmatrix} \dfrac{W}{2} & 0 & 0 & x_0 + \dfrac{W}{2} \\[10pt] 0 & -\dfrac{H}{2} & 0 & y_0 + \dfrac{H}{2} \\[10pt] 0 & 0 & \dfrac{f_d - n_d}{2} & \dfrac{f_d + n_d}{2} \\[6pt] 0 & 0 & 0 & 1 \end{bmatrix}$

注意 \(y\) 那一行的係數是 \(-H/2\)（負號表示翻轉），\(z\) 那一行和 x 方向一樣只是平移+縮放。 對於最常見的情況（\(x_0=0, y_0=0, n_d=0, f_d=1\)）：

$M_{\text{viewport}} = \begin{bmatrix} W/2 & 0 & 0 & W/2 \\ 0 & -H/2 & 0 & H/2 \\ 0 & 0 & 1/2 & 1/2 \\ 0 & 0 & 0 & 1 \end{bmatrix}$
驗證：把 NDC 角點代入確認映射正確。
\((x, y, z) = (+1, +1, +1)\)（NDC 右上前）：
\(\quad x_s = W/2 \cdot 1 + W/2 = W\)，\(y_s = -H/2 \cdot 1 + H/2 = 0\)，\(z_s = 1/2 \cdot 1 + 1/2 = 1\)。
結果 \((W, 0, 1)\)——螢幕右上角、深度最大 ✓

\((x, y, z) = (-1, -1, -1)\)（NDC 左下後）：
\(\quad x_s = 0\)，\(y_s = H\)，\(z_s = 0\)。結果 \((0, H, 0)\)——螢幕左下角、深度最小 ✓

5
C++ 實作

// 建構 Viewport 矩陣 // x0, y0 : 視口左上角（通常為 0, 0） // w, h : 視口寬高（像素） // near_d : 深度緩衝 near（通常 0.0f） // far_d : 深度緩衝 far（通常 1.0f） Mat4 viewport_matrix(float x0, float y0, float w, float h, float near_d = 0.0f, float far_d = 1.0f) { float hw = w * 0.5f; float hh = h * 0.5f; float hd = (far_d - near_d) * 0.5f; Mat4 m; m[0] = { hw, 0, 0, x0 + hw }; m[1] = { 0, -hh, 0, y0 + hh }; // 負號：y 翻轉 m[2] = { 0, 0, hd, (far_d + near_d) * 0.5f }; m[3] = { 0, 0, 0, 1.0f }; return m; }

在軟體光柵化管線中，視口矩陣通常直接以函數計算螢幕座標，而不是顯式用矩陣相乘：

// 等效的直接計算（不建立矩陣） Vec3 ndc_to_screen(Vec3 ndc, int width, int height) { return { (ndc.x + 1.0f) * 0.5f * width, (1.0f - ndc.y) * 0.5f * height, // 1−y：翻轉 (ndc.z + 1.0f) * 0.5f // z 重映射到 [0,1] }; }

6
像素中心的處理

計算出的螢幕座標是連續的浮點數；光柵化時需要決定它對應哪些像素。 常見慣例：像素 \((i, j)\) 的中心在 \((i + 0.5,\; j + 0.5)\)。

OpenGL / DirectX 慣例
像素 \((i, j)\) 覆蓋 \([i, i+1) \times [j, j+1)\)，
中心在 \((i + 0.5, j + 0.5)\)。

光柵化時通常在像素中心取樣判斷是否在三角形內（Ch09 Edge Function）。
整數截斷 vs 四捨五入
把浮點螢幕座標 \(x_s\) 轉成像素整數索引時：
通常用 (int)x_s（截斷），而不是 round(x_s)，
因為截斷等效於「落在哪個 \([i, i+1)\) 區間」的判斷。

7
完整管線：NDC 到螢幕的每一步

在軟體光柵化中，這個流程通常在 vertex shader 之後逐頂點執行一次， 得到三角形三個頂點的螢幕座標，再呼叫 DrawTriangle（Ch09–Ch12）進行光柵化。

// 完整頂點變換流程（概念示意） for (Vertex& v : triangle) { // 1. MVP 變換 Vec4 clip = uProjection * uView * uModel * Vec4(v.pos, 1.0f); // 2. 透視除法 → NDC Vec3 ndc = { clip.x / clip.w, clip.y / clip.w, clip.z / clip.w }; // 3. 視口變換 → 螢幕座標 v.screen_x = (1 + ndc.x) * 0.5f * width; v.screen_y = (1 - ndc.y) * 0.5f * height; // y 翻轉 v.depth = (1 + ndc.z) * 0.5f; }
本章重點總結
  • NDC 的 \([-1,1]^3\) 是與解析度無關的標準空間；視口變換把它映射到像素座標
  • X 映射：\(x_s = (x_{\text{ndc}}+1)/2 \cdot W\)；Y 映射有 y 翻轉：\(y_s = (1-y_{\text{ndc}})/2 \cdot H\)
  • y 翻轉原因：NDC y 朝上，螢幕 y 朝下（左上角為原點）
  • Z 重映射到深度緩衝範圍 \([0,1]\)：\(z_s = (z_{\text{ndc}}+1)/2\)
  • viewport_matrix 的 \(y\) 行係數是 \(-H/2\)，這個負號就是 y 翻轉的來源

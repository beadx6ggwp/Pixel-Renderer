2 第 55–58 行：包圍盒（Bounding Box）

min_x = min(v1.x, v2.x, v3.x) max_x = max(v1.x, v2.x, v3.x)
min_y = min(v1.y, v2.y, v3.y) max_y = max(v1.y, v2.y, v3.y)

畫三角形最直觀的方式：找到一個能「框住」三角形的軸對齊矩形， 然後對矩形內的每個像素做點在三角形內的測試。 在矩形外的像素不可能在三角形內，所以不需要測試。

為什麼先取整（int）再取 min/max？
頂點座標是 float（未來支援次像素精度），但像素座標必須是整數。 截斷取整（而非四捨五入）確保包圍盒不會比三角形小—— 例如 v1.x = 3.9，截斷後 min_x = 3，包含了 x=3 這一列，不會漏掉像素。

3 第 61–63 行：裁切到畫面邊界

min_x = max(min_x, 0) max_x = min(max_x, width - 1)
min_y = max(min_y, 0) max_y = min(max_y, height - 1)

三角形可能部分超出畫面範圍（頂點在螢幕外）。 把包圍盒和畫面矩形取交集，避免後面呼叫 SetPixel 時越界 （雖然 SetPixel 內部有邊界檢查，但提前裁切可以省掉不必要的像素測試）。

4 第 71–77 行：重心座標（Barycentric Coordinates）

對包圍盒內的每個像素 (x, y)，計算它相對於三角形的重心座標 (w1, w2, w3)：

// 共用分母（等於三角形面積的 2 倍，有向） D = (v2.y - v3.y)·(v1.x - v3.x) + (v3.x - v2.x)·(v1.y - v3.y) w1 = [(v2.y - v3.y)·(x - v3.x) + (v3.x - v2.x)·(y - v3.y)] / D w2 = [(v3.y - v1.y)·(x - v3.x) + (v1.x - v3.x)·(y - v3.y)] / D w3 = 1 - w1 - w2
這個公式從哪裡來？
重心座標 w1 是「點 P 和邊 v2-v3 組成的子三角形面積」除以「整個三角形面積」。 用叉積計算面積（2D 叉積 = 行列式），整理後就是上面的公式。 分母 D 是整個三角形的帶符號面積 × 2，是常數，只需計算一次。 （目前程式碼重複計算了分母，是效能最佳化空間。）

為什麼三個係數都 ≥ 0 就表示點在三角形內？

重心座標的幾何意義：w1、w2、w3 是點 P 到三條邊的「相對位置」。 當所有係數都在 [0, 1] 區間（且 w1+w2+w3=1）， 點 P 可以表示為三個頂點的凸組合，意思是它在三角形內部。 任何一個係數為負表示 P 在對應邊的外側。

重心座標的用途不只是「點在內？」
w1、w2、w3 同時也是插值係數。任何頂點屬性（顏色、UV、法向量、深度） 都可以用 w1·a1 + w2·a2 + w3·a3 插值到像素位置—— 這就是後面加入顏色插值（Ch10）和紋理貼圖的基礎。

5 目前的限制與下一步

目前版本的限制：
· 無 Z-buffer：多個三角形重疊時，後畫的覆蓋先畫的（畫家算法）
· 純色填充：所有像素用同一個 color，無法做顏色插值或貼圖
· 無透視修正：未來加入 3D 投影後，直接線性插值 UV 會有扭曲
· 效能：分母 D 在內層迴圈重複計算（應提到外層）

這些限制分別對應後面的實作章節：

✓ Clear() + SetPixel() + DrawLine() + DrawTriangle()（基礎版）
⬜ Z-buffer（對應理論 Ch11）
⬜ 顏色插值（對應理論 Ch10）
⬜ DrawTriangle(IShader&)：透視正確插值（對應理論 Ch24）

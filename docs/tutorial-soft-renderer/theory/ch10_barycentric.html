PART 3 · CHAPTER 10
重心座標（Barycentric Coordinates）
Edge Function 的值 ÷ 總面積，就是插值係數
barycentric interpolation varyings area ratio

上一章我們算出了三個 Edge Function 值 E0、E1、E2，知道「點在不在三角形內」。 但我們還缺一件事：在三角形內的點要填什麼顏色？

三個頂點各有自己的顏色，中間的像素顏色是三者的混合—— 離哪個頂點近，就偏向那個顏色。這個「混合的比例」就是重心座標。

1
從 Edge Function 直接推導

回顧：E0(P) 是 cross(V1−V0, P−V0)，它的絕對值等於 以 V0V1 為底邊、P 為頂點的子三角形面積的兩倍。

定義三個重心座標：

令 \(S\) = 三角形 \(V_0V_1V_2\) 的有向面積 \(= \tfrac{1}{2}\,\text{cross}(\mathbf{V}_1-\mathbf{V}_0,\;\mathbf{V}_2-\mathbf{V}_0)\)

$\begin{array}{rll} \textcolor{#a6e3a1}{w_0} &= \dfrac{\text{Area}(\triangle P V_1 V_2)}{S} & \text{（T₀ 面積比，對應 V₀）}\\[10pt] \textcolor{#fab387}{w_1} &= \dfrac{\text{Area}(\triangle V_0 P V_2)}{S} & \text{（T₁ 面積比，對應 V₁）}\\[10pt] \textcolor{#89b4fa}{w_2} &= \dfrac{\text{Area}(\triangle V_0 V_1 P)}{S} & \text{（T₂ 面積比，對應 V₂）} \end{array}$

子三角形的面積就是 Edge Function 的值的一半，所以：

子三角形面積 = Edge Function 值的一半，其中 \(E_{\text{total}} = \text{cross}(\mathbf{V}_1-\mathbf{V}_0,\;\mathbf{V}_2-\mathbf{V}_0)\)

$\textcolor{#a6e3a1}{w_0} = \dfrac{E_0(\mathbf{P})}{E_{\text{total}}},\qquad \textcolor{#fab387}{w_1} = \dfrac{E_1(\mathbf{P})}{E_{\text{total}}},\qquad \textcolor{#89b4fa}{w_2} = \dfrac{E_2(\mathbf{P})}{E_{\text{total}}}$
計算上的好消息：你在 Ch09 掃描迴圈裡已經算好了 e0、e1、e2。 只需要再除以一個常數（E_total），就直接得到重心座標——不需要額外的工作。

2
三個基本性質

性質 1
三係數之和恆為 1
w0 + w1 + w2 = 1，永遠成立。這不是巧合，而是面積分割的必然結果：三個子三角形合在一起就是整個三角形。
性質 2
P 在三角形內時，三係數都在 [0, 1] 之間
如果 P 在三角形外，至少有一個係數為負。這正是 Ch09「三值同號」判斷法的幾何解釋。
性質 3
P 在頂點時，對應係數為 1，其他為 0
P = V0 → w0=1, w1=0, w2=0。這保證了插值在頂點處等於頂點本身的值。

視覺化：係數如何隨位置變化

3
插值公式

有了 w0、w1、w2，你可以在三角形內插值任何頂點屬性：

$\text{屬性}(P) = \textcolor{#a6e3a1}{w_0}\cdot\text{屬性}(V_0) + \textcolor{#fab387}{w_1}\cdot\text{屬性}(V_1) + \textcolor{#89b4fa}{w_2}\cdot\text{屬性}(V_2)$

這個公式適用於：

  • 顏色（RGB 三個分量各自插值）
  • UV 座標（紋理映射的起點）
  • 法向量（Gouraud / Phong shading）
  • 深度值 z（Z-buffer 插值）
  • 任何你想要連續變化的數值

顏色插值的具體例子

假設三個頂點的顏色是：

V0
紅 (1, 0, 0)
V1
綠 (0, 1, 0)
V2
藍 (0, 0, 1)

重心點（w0 = w1 = w2 = 1/3）的顏色：

$\begin{array}{rcl} R &=& \tfrac{1}{3}{\cdot}1 + \tfrac{1}{3}{\cdot}0 + \tfrac{1}{3}{\cdot}0 = \textcolor{#f9e2af}{0.333}\\[6pt] G &=& \tfrac{1}{3}{\cdot}0 + \tfrac{1}{3}{\cdot}1 + \tfrac{1}{3}{\cdot}0 = \textcolor{#f9e2af}{0.333}\\[6pt] B &=& \tfrac{1}{3}{\cdot}0 + \tfrac{1}{3}{\cdot}0 + \tfrac{1}{3}{\cdot}1 = \textcolor{#f9e2af}{0.333} \end{array}$

→ 灰色（三者等比混合）

4
與 GLSL Varyings 的對應

如果你寫過 GLSL，應該看過這樣的程式碼：

// Vertex Shader
out vec3 fragColor; // 宣告 varying

void main() {
    fragColor = vertexColor; // 每個頂點設定自己的值
}

// Fragment Shader
in vec3 fragColor; // 讀取插值後的值

void main() {
    gl_FragColor = vec4(fragColor, 1.0); // 已經是插值結果
}
GPU 在中間做了什麼？
Vertex Shader 執行三次（每頂點一次），產生三個 fragColor 值。
光柵化階段：GPU 用重心座標對三個值插值。
Fragment Shader 收到的 fragColor 是已經插值好的結果。

這整個流程，就是我們這三章（Ch08-10）推導出來的數學。

命名的由來

舊版 GLSL 把這類變數叫做 varying，因為它的值在三角形各點連續變化。 現代 GLSL（1.30+）改稱 in / out，但概念完全相同。 在本系列的 IShader 設計（Part 7）中，這些插值屬性也會有自己的對應實作。

5
程式碼實作

// 假設已知：
// e0, e1, e2 ← 從 Ch09 掃描迴圈裡得到的 Edge Function 值
// e_total ← cross(V1−V0, V2−V0)，在掃描前算一次

// Step 1：計算重心座標
float inv = 1.0f / e_total;
float w0 = e0 * inv;
float w1 = e1 * inv;
float w2 = e2 * inv;

// Step 2：插值顏色（每個分量獨立插值）
float r = w0 * c0.r + w1 * c1.r + w2 * c2.r;
float g = w0 * c0.g + w1 * c1.g + w2 * c2.g;
float b = w0 * c0.b + w1 * c1.b + w2 * c2.b;

SetPixel(x, y, {r, g, b});
注意：這裡的插值是螢幕空間的線性插值，不是透視正確插值。
對平面的 2D 三角形完全正確；但對 3D 透視投影後的三角形， UV 和深度的插值需要額外的 1/w 修正（Part 7 Ch24 的主題）。
現在先用線性版本理解概念，之後再加修正。

6
本章小結

本章學到的三件事：

1. 重心座標 w0/w1/w2 = 各子三角形面積 / 總面積， 直接從 Edge Function 值除以 E_total 得到。
2. 三係數和為 1，在三角形內都非負——這正是 Ch09 內外判斷的幾何解釋。
3. 插值公式 f(P) = w0·f(V0) + w1·f(V1) + w2·f(V2) 適用於任何頂點屬性， 對應 GLSL 的 varyings 機制。

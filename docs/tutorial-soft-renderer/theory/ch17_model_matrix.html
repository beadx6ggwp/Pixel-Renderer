PART 4 · CHAPTER 17
Model 矩陣（TRS 組合）
把縮放、旋轉、平移組合成一個矩陣，把物件從物件空間送進世界空間
Model 矩陣 TRS 組合 物件→世界 非可換性 法向量矩陣

Ch16 說明了 T、R、S 都可以寫成 \(4\times4\) 矩陣。 現在要把它們組合成 Model 矩陣 M， 完成渲染管線的第一步：把物件從自身的座標系放進世界。

1
物件空間與世界空間

建模時，藝術家在以物件原點為中心的座標系裡工作——這叫物件空間（Object Space）。 一個球的頂點可能都在原點附近，X 範圍 \([-1,1]\)。

場景裡的多個物件需要放進同一個世界座標系（World Space）。 每個物件有自己的位置、朝向、大小：

物件 縮放 旋轉 平移（世界位置）
角色 A 均勻 ×2 Y 軸 45° (10, 0, 5)
角色 B 均勻 ×1 Y 軸 −90° (−3, 0, 2)
地板 (100, 1, 100) 無 (0, −1, 0)

每個物件有自己的 Model 矩陣 \(M\)，頂點乘上 \(M\) 就從物件空間移到世界空間：

$\mathbf{v}_{\text{world}} = M \cdot \mathbf{v}_{\text{object}}$

2
TRS 標準順序

Model 矩陣通常由三個基本變換組合而成，標準順序稱為 TRS：

$M = T \cdot R \cdot S$
第 1 步（最先執行）
S
縮放
調整物件大小
→
第 2 步
R
旋轉
繞物件自身軸旋轉
→
第 3 步（最後執行）
T
平移
移到世界位置
矩陣乘法是從右往左執行的。
\(M\mathbf{v} = (T \cdot R \cdot S)\,\mathbf{v} = T\bigl(R\bigl(S\,\mathbf{v}\bigr)\bigr)\)
寫成 \(M = T \cdot R \cdot S\)，執行順序仍然是先 S，再 R，最後 T。

為什麼是這個順序？

直觀理由：物件要先「整形」再「定位」。 旋轉應該繞物件自身的軸旋轉，而不是世界原點。 如果先平移再旋轉，物件會繞世界原點旋轉——通常不是想要的。

3
順序不可交換：具體反例

設平移 \(t_x = 3\)，繞 Z 軸旋轉 \(90°\)，對頂點 \(\mathbf{p} = (1, 0, 0, 1)^T\) 計算兩種順序：

✓ 正確：先 S·R，後 T（TRS 順序）

先旋轉，讓物件朝向正確方向：

$R_{90°}\begin{pmatrix}1\\0\\0\\1\end{pmatrix} = \begin{pmatrix}0\\1\\0\\1\end{pmatrix}$

再平移到世界位置：

$T\begin{pmatrix}0\\1\\0\\1\end{pmatrix} = \begin{pmatrix}3\\1\\0\\1\end{pmatrix}$

結果：(3, 1, 0)——物件在 (3,0,0) 並且已旋轉

✗ 錯誤：先 T，後 R（反序）

先平移：

$T\begin{pmatrix}1\\0\\0\\1\end{pmatrix} = \begin{pmatrix}4\\0\\0\\1\end{pmatrix}$

再旋轉——但這是繞世界原點旋轉：

$R_{90°}\begin{pmatrix}4\\0\\0\\1\end{pmatrix} = \begin{pmatrix}0\\4\\0\\1\end{pmatrix}$

結果：(0, 4, 0)——物件跑到完全不同的位置

結論：矩陣乘法不可交換，\(T \cdot R \neq R \cdot T\)。
先旋轉後平移（TRS 順序）：旋轉繞物件自身原點，平移才把它送到世界位置。
先平移後旋轉：旋轉中心變成世界原點，物件會「公轉」而非「自轉」。

4
完整 Model 矩陣展開

把 \(T(t_x,t_y,t_z)\)、\(R_y(\theta)\)、\(S(s_x,s_y,s_z)\) 相乘得到一個 \(4\times4\) 矩陣：

$M = T \cdot R_y(\theta) \cdot S = \begin{pmatrix} s_x\cos\theta & 0 & s_z\sin\theta & t_x \\ 0 & s_y & 0 & t_y \\ -s_x\sin\theta & 0 & s_z\cos\theta & t_z \\ 0 & 0 & 0 & 1 \end{pmatrix}$

幾個觀察：

  • 左上 \(3\times3\) 子矩陣包含了縮放與旋轉的資訊（線性部分）
  • 最右欄的前三行是平移向量 \((t_x,t_y,t_z)\)（仿射部分）
  • 最下列永遠是 \((0,0,0,1)\)（齊次座標的不變式）

在程式中建立 Model 矩陣

// 建立各分量 mat4 S = scale(sx, sy, sz); mat4 R = rotate_y(theta); mat4 T = translate(tx, ty, tz); // 組合（TRS 順序：從左到右寫，執行時從右往左） mat4 M = T * R * S; // 送進 shader（uniform 變數） glUniformMatrix4fv(model_loc, 1, GL_FALSE, &M[0][0]);

5
法向量的特殊情況

頂點乘上 Model 矩陣 \(M\) 就能正確變換位置。 但法向量不能直接乘 \(M\)——非均勻縮放（sx≠sy）會使法向量不再垂直於表面。

法向量矩陣（Normal Matrix）

正確的法向量變換矩陣是 \(M\) 的逆矩陣的轉置，稱為 Normal Matrix：

$\mathbf{N}' = \left(M^{-1}\right)^T \mathbf{N}$
推導直覺：法向量 \(\mathbf{N}\) 與切線 \(\mathbf{T}\) 垂直， 即 \(\mathbf{N} \cdot \mathbf{T} = 0\)。 切線用 \(M\) 變換後變成 \(M\mathbf{T}\)。 要讓新法向量 \(\mathbf{N}'\) 仍垂直於 \(M\mathbf{T}\)， 需要 \(\mathbf{N}' \cdot M\mathbf{T} = 0\)， 展開可得 \(\mathbf{N}' = (M^{-1})^T\mathbf{N}\)。

特殊情況：可以直接用 M 的旋轉部分

如果 Model 矩陣只包含均勻縮放（sx=sy=sz=s）和旋轉， 則 \((M^{-1})^T\) 的旋轉部分等於 \(R\) 本身（因為旋轉矩陣是正交的，\(R^{-1}=R^T\)）。 此時法向量可以直接乘旋轉矩陣，不需要計算逆矩陣。

只有包含非均勻縮放或切變時，才必須用完整的 Normal Matrix。

6
Model 矩陣在 Shader 中

在 OpenGL/WebGL 的 vertex shader 裡，Model 矩陣作為 uniform 傳入， 每個頂點都做相同的矩陣乘法：

// Vertex Shader (GLSL) uniform mat4 uModel; // Model 矩陣 uniform mat3 uNormalMatrix; // (M⁻¹)ᵀ 的左上 3×3 in vec3 aPosition; // 物件空間頂點 in vec3 aNormal; // 物件空間法向量 void main() { // 頂點：物件空間 → 世界空間 vec4 worldPos = uModel * vec4(aPosition, 1.0); // 法向量：用 Normal Matrix 修正（方向向量 w=0） vec3 worldNormal = normalize(uNormalMatrix * aNormal); }
本章重點總結
  • Model 矩陣 \(M = T \cdot R \cdot S\) 把頂點從物件空間變換到世界空間
  • 執行順序：先 S（縮放），再 R（旋轉），最後 T（平移）——矩陣從右往左
  • 矩陣乘法不可交換；先旋轉後平移是繞物件自身軸旋轉，反之是繞世界原點
  • 每個物件有自己的 Model 矩陣，CPU 算一次，GPU 並行套用到所有頂點
  • 法向量不能直接乘 M；非均勻縮放需要 Normal Matrix \((M^{-1})^T\)
